# S1 纯卖权策略 2026 年 5 月工作计划

文档日期：2026-04-28  
策略对象：S1 日频纯卖权策略  
阶段目标：研究版策略推进到模拟盘验证版  
重要边界：2026 年 5 月目标是上模拟盘，不是上实盘。

## 1. 管理层摘要

2026 年 5 月的核心目标不是继续做零散参数优化，而是把 S1 纯卖权策略从研究脚本推进到可以进行模拟盘验证的策略雏形。月末应形成一套可复现、可解释、可监控、可风控评估的策略版本，并同步完成 IT 数据与系统需求、容量评估、压力测试和模拟盘运行流程。

S1 的策略定位保持不变：

```text
以卖出低 Delta、次月场内期权为主要结构，
通过收取权利金捕捉波动率风险溢价和 theta，
同时用止损、异常报价过滤、保证金控制、品种分散和压力测试控制尾部风险。
```

5 月不以实盘交易为目标。模拟盘阶段主要回答以下问题：

- 策略信号是否稳定、可复现。
- 开仓和平仓规则是否能被系统每日自动生成。
- 数据链路是否完整、及时、可审计。
- 回测表现是否能在模拟盘信号中保持一致的风险画像。
- 容量、保证金、止损聚集和极端情景风险是否可被日常监控。

## 3. 当前研究基础

截至 2026-04-28，项目已经完成以下基础工作：

### 3.1 策略逻辑与外部调研

- 已围绕乐得同类卖权策略做过多轮净值画像和持仓结构研究，重点观察其保证金使用率、P/C 结构、品种分散、持仓手数、回撤修复路径和收益来源。
- 已与乐得相关人员沟通并确认若干核心交易直觉：该类策略本质不是盘中频繁 Delta/Gamma 对冲，而是卖较虚值、低 Delta、次月附近期权，收取权利金，并通过权利金止损和不扛单控制尾部。
- 已将止损规则从较早的 1.5x/1.8x 实验扩展到 2.5x 权利金止损，并加入异常跳价确认，避免低流动性深虚值期权的瞬时异常价触发伪止损。
- 已确认低 Delta 是核心硬约束之一。当前主线仍坚持 `abs(delta) <= 0.10`，不通过卖更高 Delta 来换取表面权利金。
- 已讨论并验证过保护腿、裸卖、双卖、单边卖、P/C 偏移、持有到期、提前止损、临近到期处理等结构问题。当前研究主线回到更干净的纯卖权框架，再逐步加入排序、预算和风控。
- 已明确 5 月目标不是上实盘，而是形成可进入模拟盘验证的 S1 纯卖权策略版本。

### 3.2 研究范式变化

- 研究方式已经从早期“直接调参数、看 NAV”转向“先建立标准基准，再做因子分层和候选池 shadow label 检验”。
- 已认识到不能只在实际成交样本中验证 B2/B3 因子，因为成交样本已经受到策略排序和预算倾斜影响，存在处理后检验偏差。
- 已建立 full shadow candidate universe 框架：在 B2/B3 倾斜之前的 B1 可交易候选池上，给所有候选合约同时打 B2/B3 因子和未来 1 手 shadow outcome 标签。
- 已将后续研究重点从“哪个参数历史收益最高”转向“哪些因子能解释权利金留存、止损概率、vega/gamma 损耗和尾部亏损”。
- 已明确因子的不同用途：部分适合做合约排序，部分适合做品种-方向预算倾斜，部分只适合作为风险惩罚或模拟盘监控指标。
- 已将策略目标从单纯提升收益扩展为：收益接近目标、最大回撤可控、vega 归因改善、权利金留存率提高、止损聚集可解释、容量可评估。

### 3.3 回测与工程基础

- B0 基准：全品种、次月、低 Delta、持有到期或 2.5x 权利金止损的纯卖权基准。
- B1：在 B0 基础上加入成交量和持仓量排序，提升可交易性。
- B2：引入权利金质量因子，包括 IV/RV carry、权利金对 IV shock 覆盖、权利金对 stress loss 覆盖、theta/vega 效率、gamma rent 惩罚和交易摩擦。
- B2c：目前是较强的主线版本之一，显示出更高权利金收入和较好的留存，但 vega/gamma 损耗仍未彻底解决。
- B3a-e：正在测试 forward variance、vol-of-vol、IV shock coverage、vomma penalty 和 clean vega composite 等增强因子。
- Full shadow candidate universe：已重新启动 B2/B3 全因子候选池标签实验，用于在 B2/B3 倾斜前样本上检验因子真实有效性。
- 报告系统：已具备 NAV、回撤、保证金、Greeks、PnL 归因、品种结构、P/C 结构等图表和报告输出能力。
- 已修正和审查过多项工程口径问题，包括真实标的价格映射、PCP spot 估计口径、Black76 估值、手续费和保证金参数、低价合约过滤、ETF 数据映射问题、分钟级止损效率优化、日内最高价预筛止损等。
- 已建立本地、GitHub、服务器三端同步流程，并逐步将策略规则、实验设计、报告输出和配置文件文档化。

## 4. 关键判断口径

5 月的目标不是选出历史回测收益最高的一组参数，而是建立一套可以进入模拟盘观察的策略系统。

核心判断标准如下：

| 指标 | 目标或要求 |
| --- | --- |
| 年化收益 | 中期目标靠近 6%，但 V1 更重视逻辑正确、可复现和风险可控 |
| 最大回撤 | 目标小于 2%；如果阶段性超过，需要清楚解释来源和可控性 |
| Vega 归因 | 目标逐步改善，不能长期主要依赖方向收益或 residual |
| Theta 留存 | 必须稳定跟踪权利金收入、权利金留存率和止损损耗 |
| 止损聚集 | 必须监控同一日期、同一板块、同一品种连续止损 |
| 容量 | 必须用真实成交量、持仓量和参与率估算，不只看回测 NAV |
| 可交易性 | 必须考虑手续费、滑点、低价合约、异常报价、分钟级止损确认 |
| 可复现性 | 策略配置、数据版本、运行命令、输出报告必须可复现 |
| 模拟盘可用性 | 每日能自动生成信号、模拟成交、风险检查和复盘报告 |

### 7.2 容量分析要求

容量评估至少包括：

- 单合约成交量参与率。
- 单合约持仓量占比。
- 单品种最大可承载手数。
- 商品、ETF、股指分市场容量。
- 1000 万、3000 万、1 亿、3 亿资金规模下的可交易性。
- 流动性折价和滑点敏感性。
- 止损时的退出容量，而不只看开仓容量。

### 7.3 压力测试要求

压力测试至少包括：

| 场景 | 内容 |
| --- | --- |
| IV shock | 全组合 IV +3vol、+5vol、+10vol |
| Spot shock | Put 侧标的 -3%、-5%、-8%；Call 侧标的 +3%、+5%、+8% |
| 联合冲击 | 标的不利移动 + IV 上升 + skew steepening |
| 流动性冲击 | 平仓滑点扩大、低流动性合约无法即时成交 |
| 止损聚集 | 同日多品种触发 2.5x 权利金止损 |
| 保证金冲击 | 保证金率上调、标的剧烈波动导致保证金占用上升 |

### 7.4 验收标准

第 3 周结束时，需要能回答：

- 如果资金从 1000 万放大到 1 亿或 3 亿，策略是否还能成交。
- 哪些品种最容易形成容量瓶颈。
- 哪些压力场景会导致最大亏损。
- 当前策略最先触发的是保证金约束、流动性约束、vega/gamma 约束还是止损聚集约束。
- 模拟盘阶段需要重点监控哪些风险指标。

## 10. 风险边界

模拟盘阶段必须明确以下风险边界：

- 不进入实盘交易。
- 不自动下单。
- 不使用未来数据修正信号。
- 不用回测后的最佳结果反向修改当天信号。
- 不因短期模拟盘表现好就跳过容量和压力测试。
- 不把低 Delta 误解为无风险。
- 不把高权利金误解为高质量权利金。
- 不只看 NAV，需要同时看 vega/gamma 损耗、止损聚集和权利金留存。
