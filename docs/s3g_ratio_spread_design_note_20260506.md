- `估值表日报-SXB208-国优卷柏稳健对冲10号私募证券投资基金-4-20260310.xlsx`

已完成的数据库校验与 Greek 计算输出：

Greek 口径：

- Vega 口径为每 1 vol point。
- Theta 口径为每日 theta。

## 2. 管理人结构画像

估值表显示，该管理人的持仓不是简单的一买二卖比例价差，而是更接近“多档比例价差”：

```text
买入一个或多个较近执行价期权
+ 卖出多个更远执行价期权
= 用远端卖腿权利金融资近端买腿，同时保留局部方向收益与局部 long gamma
```

其主要特征包括：

- 买腿更靠近平值，Delta 明显高于卖腿。
- 卖腿更远虚值，单腿 Delta 通常较低。
- 卖腿数量明显多于买腿，常见比例约 `1:3` 到 `1:8`，个别结构更高。
- 结构常接近净权利金收入或低成本建立，而不是单纯支付权利金买方向。
- 标的多为商品期权，尤其是黄金、白银、玻璃、纯碱、原油、白糖、碳酸锂等。

## 3. Delta 分布结论

从已计算的两张估值表看，管理人真正卖出的腿并不是高 Delta。

### 2026-03-10

```text
买 Call：中位 Delta 约 0.15，范围大约 0.04-0.37
卖 Call：中位 Delta 约 0.035，最大约 0.088

买 Put：中位 Delta 约 0.08，范围大约 0.005-0.185
卖 Put：中位 Delta 约 0.011，最大约 0.059
```

### 2025-09-01

```text
买 Call：中位 Delta 约 0.05，但黄金买腿约 0.44，玻璃买腿因使用主连 fallback 需要谨慎看待
卖 Call：中位 Delta 约 0.012，最大约 0.095

Put 腿较少：
买 Put 约 0.043
卖 Put 约 0.005
```

### 典型结构

```text
AG2604 Call：买 0.091 delta，卖 0.027 delta，比例约 1:4
AU2604 Call：买 0.122 delta，卖 0.025 delta，比例约 1:5
SA605 Call：买 0.183 delta，卖 0.059 delta，比例约 1:2.5
SR605 Call：买 0.276/0.372 delta，卖 0.088 delta，比例约 1:2
AU2510 Call：买 0.438 delta，卖 0.025 delta，比例约 1:8
LC2605 Put：买 0.112 delta，卖 0.024 delta，比例约 1:4.4
```

核心结论：

```text
买腿 Delta：常见 0.10-0.30，个别更高
卖腿 Delta：多数 0.01-0.08，硬上限大致不超过 0.10
```

因此，不能把该结构理解为“卖 0.2 delta 的比例价差”。更准确的理解是：

```text
买中低 Delta 的近端腿，拿局部 gamma 和方向弹性；
卖多个极远端低 Delta 腿，收权利金、融资买腿、承担极端尾部风险。
```

## 4. 策略本质

该结构不是纯卖权，也不是纯方向多头，而是：

```text
方向判断 + 波动融资 + 局部 long gamma + 远端尾部风险承接
```

收益来源可以拆成：

```text
1. 近端买腿在小到中等幅度方向移动中获利。
2. 近端买腿提供局部 long gamma，行情慢慢靠近买腿时收益非线性改善。
3. 远端卖腿提供 theta 与权利金，覆盖买腿成本并争取额外净收入。
4. 如果 IV 回落，结构整体 short vega 的部分可以贡献收益。
5. 如果行情没有大幅穿透远端卖腿，到期时卖腿大概率归零，权利金留存。
```

主要风险是：

```text
1. 标的发生大涨或大跌，穿透远端卖腿。
2. 卖腿 Delta 快速抬升，局部 long gamma 被远端多倍 short gamma 反噬。
3. IV 与 skew 同时上升，远端卖腿价格大幅扩张。
4. 卖腿比例过高，穿透后的 tail slope 太陡。
5. 极低价卖腿受 tick 和流动性影响，止损和估值可能不稳定。
```

## 5. 与当前 S3 的差距

当前代码中的 S3 更像简化版二腿比例价差：

```text
买入一个较近腿
+ 卖出一个较远腿
比例通常在 1:2 或 1:3 附近
```

与管理人结构相比，当前 S3 主要缺口是：

- 不支持多执行价买腿和多执行价卖腿。
- 比例上限偏低，无法表达 `1:4`、`1:6`、`1:8` 这类融资结构。
- 没有显式使用买腿 Delta、卖腿 Delta、净权利金、盈亏平衡点、tail slope 来共同定价结构。
- 没有对“局部 long gamma 区间”和“远端尾部亏损区间”做单独约束。
- 保护腿或转断翅蝶逻辑尚未成为主路径。

因此，后续建议不要直接修改原 S3，而是新增或扩展为 S3G：

```text
S3G = Guoyou-style multi-strike ratio spread
```

## 6. 推荐的 S3G 参数框架

### 保守版

```text
买腿 Delta：0.10-0.20
卖腿 Delta：0.02-0.06
卖腿硬上限：0.08
比例：1:3 到 1:6
净权利金：>= 0，或卖腿权利金 / 买腿成本 >= 1.1
```

适用环境：

- 方向判断有一定置信度，但不希望承担过强尾部。
- IV 不低，远端 skew 或 wing premium 足够。
- 标的不处于高波动突破状态。

### 标准版

```text
买腿 Delta：0.12-0.25
卖腿 Delta：0.02-0.08
卖腿硬上限：0.10
比例：1:3 到 1:6
卖腿权利金 / 买腿成本：>= 1.2
```

适用环境：

- 有明确“不大涨”或“不大跌”的方向判断。
- 结构可以做到低成本或净收入。
- 盈亏平衡点距离足够远。

### 进攻版

```text
买腿 Delta：0.20-0.35
卖腿 Delta：0.04-0.09
卖腿硬上限：0.10
比例：1:2 到 1:4
```

适用环境：

- 方向判断更强。
- 买腿更接近方向交易，卖腿只是融资。
- 必须更严格控制 tail slope。

### 高权利金融资版

```text
买腿 Delta：0.05-0.12
卖腿 Delta：0.005-0.03
比例：1:6 到 1:10
```

适用环境：

- 远端 wing premium 明显偏贵。
- 卖腿极远虚值，但数量很多。
- 必须加入极严格的流动性、tick、盈亏平衡点和压力亏损约束。

## 7. 必须加入的结构约束

S3G 不应只按 Delta 开仓，必须同时检查以下指标。

### 7.1 净权利金约束

```text
net_credit = short_premium - long_cost
```

建议：

```text
基础要求：net_credit >= 0
更稳健要求：short_premium / long_cost >= 1.1-1.5
```

如果净支出过高，则该结构更像方向期权多头，不再是“比例价差融资结构”。

### 7.2 盈亏平衡点距离

对 Call ratio：

```text
upper_breakeven = short_strike + max_profit / net_short_contracts
breakeven_distance = upper_breakeven / spot - 1
```

对 Put ratio：

```text
lower_breakeven = short_strike - max_profit / net_short_contracts
breakeven_distance = 1 - lower_breakeven / spot
```

建议要求：

```text
breakeven_distance > 过去 60 日同方向最大不利移动的 90%-95% 分位
```

### 7.3 Tail slope 约束

穿透卖腿后的亏损斜率必须可控：

```text
tail_slope = net_short_contracts * multiplier * underlying_price_sensitivity
```

建议：

```text
单结构 tail stress loss <= 单笔风险预算
单品种 tail stress loss <= 单品种风险预算
组合 tail stress loss <= 组合风险预算
```

### 7.4 卖腿 Delta 演化约束

开仓时卖腿 Delta 低，不代表持仓期间安全。

建议：

```text
开仓卖腿 Delta <= 0.08，硬上限 0.10
预警：卖腿 Delta > 0.10
减仓：卖腿 Delta > 0.12
强风控：卖腿 Delta > 0.15
```

### 7.5 极低价与流动性约束

远端卖腿往往价格很低，容易出现 tick 与流动性问题。

建议：

```text
卖腿价格 >= 最小价格阈值
tick_value_ratio = minimum_tick / option_price 不得过高
成交量和持仓量必须满足退出容量
卖腿止损成交压力必须纳入压力测试
```

## 8. 交易方向与 P/C 选择

S3G 必须有方向判断，但方向判断不应过度精细。

更合理的做法是：

```text
上涨/偏强环境：优先做 Call ratio，即买近端 Call、卖远端 Call，但必须判断“不大涨”
下跌/偏弱环境：优先做 Put ratio，即买近端 Put、卖远端 Put，但必须判断“不大跌”
震荡环境：可以双侧小仓，或者只做 skew 更贵的一侧
高波突破环境：不做或只做 defined-risk 版本
```

这里的关键不是预测大方向，而是判断：

```text
标的是否可能发生极端单边穿透。
```

## 9. 与 S1 的关系

S1 是纯卖权策略，主要收益来自：

```text
Premium Pool × Deployment Ratio × Retention Rate - Tail/Stop Loss - Cost/Slippage
```

S3G 的收益结构不同：

```text
局部方向收益
+ 局部 long gamma 收益
+ 卖腿 theta / vega 收益
- 买腿时间价值损耗
- 远端 tail loss
- 交易成本和止损滑点
```

S3G 与 S1 的相关性判断：

- 平稳或慢趋势市场中，S3G 与 S1 可能同赚 theta 和 IV 回落，相关性偏正。
- 标的小幅向买腿方向移动时，S3G 可能明显优于 S1，因为买腿提供局部 gamma。
- 极端行情穿透卖腿时，S3G 与 S1 可能一起亏，且 S3G 因比例放大可能有更强尾部。
- 因此 S3G 不是天然分散器，而是一个“局部增强 + 尾部受控”的策略模块。

## 10. 后续回测建议

建议分阶段推进。

### Phase 1：复刻管理人结构画像

目标：

```text
用历史期权链每天生成类似管理人的 S3G 候选结构。
```

输出：

- 买腿 Delta 分布。
- 卖腿 Delta 分布。
- 比例分布。
- 净权利金分布。
- 盈亏平衡点距离分布。
- Tail slope 分布。
- Theta/Vega/Gamma 分布。

### Phase 2：保守版 S3G 回测

建议初始参数：

```text
买腿 Delta：0.12-0.25
卖腿 Delta：0.02-0.08
卖腿硬上限：0.10
比例上限：1:6
net_credit >= 0
short_premium / long_cost >= 1.1
breakeven_distance > 60 日不利移动 90% 分位
```

退出：

```text
卖腿权利金 2.5x 硬止损
卖腿 Delta > 0.12 减仓
卖腿 Delta > 0.15 强制平仓或转保护
到期前 2-5 天处理，避免最后 gamma 爆炸
```

### Phase 3：多档铺开

在 Phase 2 稳定后，再允许：

```text
多个买腿执行价
多个卖腿执行价
卖腿分布在 2-5 个 strike
```

核心目标是降低单一执行价穿透风险，而不是简单提高比例。

### Phase 4：保护腿或动态转断翅蝶

研究：

```text
当卖腿 Delta 或价格达到阈值时，是否买入更远端保护腿；
或开仓时直接构建低成本 broken-wing butterfly。
```

重点检验：

- 尾部亏损是否显著下降。
- 收益是否被保护腿成本过度侵蚀。
- 实盘流动性是否支持临时补腿。

## 11. 核心判断

该管理人的 S3 变种不应被理解为“高比例裸卖远端期权”，而应被理解为：

```text
用远端低 Delta 卖腿融资近端中低 Delta 买腿，
在局部保留 long gamma 和方向弹性，
同时承接极端穿透后的尾部风险。
```

因此，我们后续开发 S3G 时，最重要的不是复制 `1:8` 这种比例，而是复制以下结构逻辑：

```text
买腿 Delta：0.10-0.30
卖腿 Delta：0.01-0.08，硬上限 0.10
净权利金：覆盖买腿成本并有安全垫
局部收益区：足够宽
远端盈亏平衡点：足够远
穿透尾部斜率：受控
流动性和止损：可执行
```

一句话总结：

```text
S3G 应该是“局部 long gamma、整体收权利金、极端尾部受控”的比例价差结构，而不是简单放大比例的卖权策略。
```
