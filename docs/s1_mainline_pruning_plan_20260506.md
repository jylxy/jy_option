# S1 主线收敛与代码瘦身计划

日期：2026-05-06

## 1. 背景

S1 过去经历了 B0/B1/B2/B3/B4/B5/B6、P3/P3B、P5/P6、full shadow、autoresearch 等多轮研究。这个过程沉淀了大量实验逻辑，但也带来了两个问题：

- 主引擎长期承载过多历史实验开关，导致代码难以审计。
- 脚本和配置文件数量过多，日常主线回测不容易确认真实依赖。

目前研究主线已经收敛到 `P3B/A0`。因此，接下来需要把代码和脚本按照主线重新分层。

## 2. 分类规则

所有代码、配置和脚本分为四类：

| 类别 | 定义 | 处理方式 |
|---|---|---|
| `mainline` | P3B/A0 当前回测直接依赖 | 保留在主路径，优先补测试和性能优化 |
| `mainline-diagnostic` | 主线不影响交易，但用于复盘、报告、审计 | 保留，但不能拖慢默认回测 |
| `experimental` | 历史或后续研究仍可能复用 | 移到 `experiments/`、`scripts/` 子目录或独立模块 |
| `deprecated` | 已被否决、被替代或只用于历史复现 | 移到 `archive/`，保留 README 和引用说明 |

## 3. 主线保留范围

P3B/A0 当前必须保留：

- B0 基准规则：全品种、次月、delta 小于 0.1、最多 5 个梯队、持有到期/止损。
- B1 可交易性排序：最低价过滤、流动性和持仓量排序。
- B2C 品种预算倾斜：`s1_b2_product_tilt_enabled` 及相关诊断。
- 乐得期限偏好：黄金双月、白银/铁矿近月、部分品种主力月。
- P5 A0 组级 1.5x 止损。
- 费用、保证金、滑点、分钟止损预筛、到期处理和 NAV 归因。

## 4. 第一阶段：文档与脚本分层

本阶段目标是不改变回测经济逻辑，只降低维护噪音。

执行动作：

1. 新增 `docs/s1_p3b_a0_mainline_spec.md`，固化主线口径。
2. 新增本计划文档，明确主线、诊断、实验和归档边界。
3. 将 `scripts/` 从单层目录改为功能子目录：
   - `scripts/analysis/`
   - `scripts/reports/`
   - `scripts/launchers/`
   - `scripts/autoresearch/`
4. 维护 `scripts/s1_cli.py` 作为统一入口，避免移动脚本后命令失效。
5. 暂不移动根目录 `config_*.json`，因为配置继承链和历史远端命令仍依赖当前路径。

## 5. 第二阶段：主引擎可达性瘦身

本阶段才开始拆 `toolkit_minute_engine.py` 中的非主线逻辑。

优先抽离：

- B5/full shadow candidate universe 字段和候选面板输出。
- B3/B4/B6 合约评分和因子扩展逻辑。
- autoresearch 或历史实验专用输出。
- falling framework 和 forward vega 过滤。

抽离原则：

- 实验逻辑进入独立模块，不再散落在主引擎长函数里。
- 默认 P3B/A0 回测不初始化、不计算、不输出这些字段。
- 历史配置仍可通过显式开关调用对应模块完成复现。

## 6. 第三阶段：性能优化

在主线逻辑边界清楚后，再做性能优化：

- 日频候选生成只保留主线必要字段。
- 分钟止损继续使用日内最高价预筛，减少分钟扫描对象。
- shadow / factor / deep diagnostics 默认关闭。
- 报告图表生成与主回测彻底解耦。
- 针对长回测增加 `lite diagnostics` 模式。

## 7. 风险控制

本轮清理禁止做以下动作：

- 不直接删除历史研究文档。
- 不删除已经用于报告的输出目录。
- 不改动 P3B/A0 的交易经济逻辑。
- 不迁移配置文件继承链，除非同时更新所有引用并做回测 smoke test。
- 不在远端仍有实验运行时重置远端代码。

## 8. 验证要求

每轮结构性改动后至少执行：

```bash
python -m py_compile src/*.py scripts/*.py scripts/**/*.py
python -m pytest tests
```

如果改动涉及交易逻辑，还需要做 P3B/A0 小样本 smoke backtest，并对比：

- NAV
- 持仓数
- 开仓权利金
- 止损次数
- 保证金使用率
- Greeks
- 成交记录

## 9. 当前判断

第一阶段可以立即执行，因为它不改变回测逻辑，只整理文档和脚本入口。

第二阶段需要按模块逐步做，每次只拆一个主题，避免主引擎和历史实验一起动导致难以定位问题。
