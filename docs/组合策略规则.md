# 三策略组合 — 策略规则

**版本**：v3.1
**日期**：2026-05-08
**状态**：V3引擎回测验证完成（含nav_now保证金修复）

---

## 一、组合架构

| 子策略 | 定位 | 保证金分配 | 收益来源 | Greeks |
|--------|------|-----------|---------|--------|
| S1 垂直价差保护卖权 | 核心层 | 独立上限25% | Theta衰减 | 负Gamma负Vega |
| S3 蝶式比例价差 | 增强层 | 独立上限25% | Gamma+Theta | 混合 |
| S4 尾部博弈 | 保险层 | 0.5%权利金/月（不占保证金） | 极端行情 | 正Gamma正Vega |

> S2波动率套利默认关闭（去掉脏数据后年化≈0%），引擎中保留代码但不启用。

**S1+S3按独立保证金上限分配**，总保证金上限50%。S4是买方策略，不占保证金，按月度权利金预算运行。

## 二、资金与保证金分配

```
总资金 = 1000万（NAV随盈亏浮动，仓位按当日NAV计算）

保证金约束（V3引擎实际执行）：
  S1保证金上限：当日NAV × 25%
  S3保证金上限：当日NAV × 25%
  组合保证金上限：当日NAV × 50%
  开仓时双重检查：个策略独立上限 + 组合总上限

S1配置：
  - 15品种，每品种每方向 NAV×2% 保证金
  - 垂直价差保护2:1 + 止盈40%重开 + IV反向(>75%降仓50%)

S3配置（S3优先开仓）：
  - 15品种，买1浅虚值 + 卖3深虚值 + 买3远端保护（蝶式）
  - 仓位按卖腿保证金约束（非买腿权利金）
  - 止盈30%重开 + IV反向(>75%降仓50%)

S4配置：
  - 4品种（中证1000、沪金、原油、沪铜）
  - 买最深虚值Put+Call，持有到期
  - 每月权利金预算 = NAV×0.5%，手数上限5手/合约
```

## 三、每日执行流程

```
15:15 盘后计算：

  Step 0: 计算当日NAV
    nav_now = CAPITAL + 累积PnL
    （所有仓位用当日NAV计算，非固定初始资金）

  Step 1: Mark-to-Market
    更新所有持仓价格
    到期合约价格归零
    prev_price → cur_price 移位

  Step 2: 各策略止盈/到期检查
    S1: 卖腿浮盈≥40% → 平仓+重开（检查保证金后）
    S3: 卖腿浮盈≥30% → 平仓+重开（检查保证金后）
    S4: 到期处理（无止盈）

  Step 3: 新开仓（S3优先）
    a) 先开S3新仓（S3资金效率更高，优先使用保证金）
       - 检查S3保证金 < 25% NAV 且 组合保证金 < 50% NAV
    b) 再开S1新仓
       - 检查S1保证金 < 25% NAV 且 组合保证金 < 50% NAV
    c) 最后开S4新仓
       - 按月度权利金预算，手数上限5

  Step 4: 汇总PnL
    daily_pnl = pnl_s1 + pnl_s3 + pnl_s4
    （各策略PnL独立计算后汇总）
```

## 四、组合风控

### 4.1 引擎内置风控（已在回测中执行）

| 规则 | 阈值 | 动作 |
|------|------|------|
| S1保证金上限 | > 25% NAV | S1暂停新开仓 |
| S3保证金上限 | > 25% NAV | S3暂停新开仓 |
| 组合总保证金 | > 50% NAV | 全部暂停新开仓 |
| S4手数上限 | 5手/合约 | 限制极端波动 |
| 止盈重开保证金检查 | 重开前检查上限 | 超限则只平不开 |
| 到期归零 | DTE=0 | 持仓价格归零避免幽灵持仓 |

### 4.2 实盘额外风控（回测未模拟）

| 规则 | 阈值 | 动作 |
|------|------|------|
| 组合Cash Vega | > 3% | S1优先减仓至Vega<2% |
| 组合月度亏损 | > 3% | 全部暂停新开仓1周 |
| 组合最大回撤 | > 5% | S1+S3各减仓50% |
| 长假前 | 假期≥3天 | S1+S3各平仓50% |
| 单品种集中度 | 单品种保证金>总保证金30% | 不再加仓该品种 |

## 五、IV分位数计算

```
对每个品种：
  1. 取当天ATM期权（moneyness 0.95-1.05，DTE 15-90天）的平均IV
  2. 计算过去252个交易日的IV序列
  3. IV分位 = percentileofscore(IV历史序列, 当前IV)
     （使用scipy.stats.percentileofscore，比min-max归一化更鲁棒）
  4. 需要至少60个交易日的数据才开始计算
```

## 六、滑点模型

V3引擎在所有开平仓时计入滑点：

| 操作 | 滑点方向 |
|------|---------|
| 卖出开仓 | 成交价 = 收盘价 × (1 - slippage) |
| 买入开仓 | 成交价 = 收盘价 × (1 + slippage) |
| 卖出平仓 | 成交价 = 收盘价 × (1 - slippage) |
| 买入平仓 | 成交价 = 收盘价 × (1 + slippage) |

统一滑点率：0.2%（所有合约类型）

## 八、已修复的历史bug（V3 vs V1/V2）

| 编号 | 问题 | 修复方案 |
|------|------|---------|
| BUG-1 | S3仓位用权利金而非保证金定手数 | 改为卖腿保证金约束 |
| BUG-2 | S4手续费只收7元应为14元 | 改为14元（开+平） |
| BUG-3 | 组合权重超100% | 改为共享资金池+独立保证金上限 |
| BUG-4 | 止盈重开未检查保证金上限 | 重开前检查个策略+组合上限 |
| BUG-5 | IV分位用min-max归一化 | 改为percentileofscore |
| BUG-6 | 所有策略未计算滑点 | 所有开平仓加0.2%滑点 |
| BUG-7 | 仓位用固定初始资金 | 改为每日NAV动态计算 |
| BUG-8 | S2卖腿保证金未计入总保证金 | 移除S2排除条件 |
| BUG-9 | 到期日无数据时价格未归零 | 到期日强制cur_price=0 |
| BUG-10 | 保证金比例检查除以CAPITAL而非nav_now | 全部改为除以nav_now，与仓位计算一致 |
