# S1 B1 流动性与持仓量排序标准口径

## 1. 文档定位

本文档只记录 B1 相对标准 B0 的变化。

B1 不是新的策略框架，也不是加入 IV/RV、趋势、降波或风控后的优化版。B1 的目的非常单一：

```text
在 B0 完全相同的卖权规则、仓位规则、止损规则和费用/保证金口径下，
只把“开仓优先级”从代码顺序 / Delta 接近度，改成成交量和持仓量优先。
```

因此，B1 是一个“可交易性优先”的基准实验，用来回答：

- B0 的结果是否受不活跃合约、低成交合约和偶发异常报价影响。
- 在不加入复杂 alpha 规则的情况下，优先卖更活跃的合约，收益和回撤会如何变化。
- ETF、股指和商品中，真实高流动性品种是否会自然获得更高进入优先级。
- 后续是否需要在 B2 再加入硬性的最低成交量和最低持仓量过滤。

## 2. 版本命名

建议版本名：

```text
strategy_version = s1_baseline_b1_liquidity_oi_rank_stop25
```

建议配置文件：

```text
config_s1_baseline_b1_liquidity_oi_rank_stop25.json
```

建议正式回测标签：

```text
s1_b1_liq_oi_rank_stop25_allprod_2022_latest
```

## 3. B1 保持不变的 B0 规则

B1 除排序外，其他规则全部沿用标准 B0。

保持不变：

- 全品种扫描。
- 次近到期：`s1_expiry_mode=nth_expiry`，`s1_expiry_rank=2`。
- 只卖虚值期权。
- 卖腿 Delta 上限仍为 `abs(delta) <= 0.10`。
- Put 和 Call 都尝试开仓，不做趋势择边。
- 品种等权。
- 品种内 P/C 预算各半。
- 每个品种每侧最多 5 个合约。
- 方向内合约预算等权。
- 目标总保证金仍为 50%。
- 真实手续费表和真实保证金率。
- 不止盈。
- 持有到期。
- 权利金上涨到开仓权利金 `2.5x` 硬止损。
- 盘中止损防跳变确认。
- 允许同品种同方向同到期继续新增仓位。

仍然不启用：

- IV/RV carry 过滤。
- IV 分位过滤。
- 降波确认。
- forward vega 过滤。
- 趋势动量 P/C 偏移。
- 板块集中度约束。
- 相关性约束。
- cash Greek 上限。
- stress score。
- stress sizing。
- 止损冷却期。
- 止损后重开规则。

## 4. B1 唯一变化：排序规则

B1 只改变两个排序层级：

```text
1. 品种扫描顺序
2. 品种内、方向内、次近到期上的合约 Top5 选择顺序
```

除此之外，不改变任何候选过滤、仓位预算、保证金计算和退出规则。

## 5. 品种扫描顺序变化

### 5.1 B0 口径

B0 的品种扫描顺序为字符串排序：

```python
sorted_products = sorted(product_frames)
```

这意味着 B0 不关心当天哪个品种更活跃。

### 5.2 B1 口径

B1 改为按品种级流动性分数降序扫描。

品种级流动性分数只使用当日收盘后已知的期权链数据，不使用未来信息。

建议公式：

```text
product_liquidity_score
= 0.5 * percentile_rank(log1p(product_eligible_volume))
+ 0.5 * percentile_rank(log1p(product_eligible_open_interest))
```

其中：

```text
product_eligible_volume
= 当日该品种、次近到期、Delta 合格、权利金合格的所有 Put/Call 候选合约成交量之和

product_eligible_open_interest
= 当日该品种、次近到期、Delta 合格、权利金合格的所有 Put/Call 候选合约持仓量之和
```

排序方式：

```text
product_liquidity_score 降序
product_eligible_open_interest 降序
product_eligible_volume 降序
product_code 升序
```

说明：

- B1 不设置 `daily_scan_top_n`，仍然扫描全部候选品种。
- 品种排序只在组合接近 50% 保证金上限、整数手数和预算不足时影响结果。
- 这样能让更活跃品种在边际预算竞争中优先进入，但不会直接改变单品种等权预算。

## 6. 合约 Top5 排序变化

### 6.1 B0 口径

B0 在每个品种、每个方向、次近到期内，先筛选：

```text
OTM
abs(delta) <= 0.10
option_close >= 0.5
权利金现金额 >= 往返手续费
```

然后按接近目标 Delta 排序：

```text
delta_dist = abs(abs(delta) - 0.10)
delta_dist 越小越靠前
```

每侧最多取前 5 个合约。

### 6.2 B1 口径

B1 保持候选过滤条件不变，但 Top5 合约选择改为流动性和持仓量优先。

建议合约级流动性分数：

```text
contract_liquidity_score
= 0.5 * percentile_rank(log1p(volume))
+ 0.5 * percentile_rank(log1p(open_interest))
```

排序方式：

```text
contract_liquidity_score 降序
open_interest 降序
volume 降序
delta_dist 升序
option_code 升序
```

解释：

- `volume` 衡量当日成交活跃度。
- `open_interest` 衡量市场承载和持续交易深度。
- `delta_dist` 只作为次级排序，用来避免在流动性接近时偏离 B0 的低 Delta 核心。
- `option_code` 作为最终 tie-breaker，保证回测可复现。

## 7. B1 不加入硬过滤

B1 暂时不设置最低成交量和最低持仓量门槛。

也就是说，以下参数仍保持 B0：

```text
s1_min_volume = 0
s1_min_oi = 0
```

原因：

```text
B1 只测试排序带来的影响。
如果同时加入最低 volume/OI 过滤，就无法区分到底是“排序改善”还是“过滤减少交易”带来的变化。
```

因此，B1 仍可能在某些品种、某些日期交易低成交或低持仓合约；但只要同一候选池中存在更活跃合约，系统会优先选择更活跃合约。

硬过滤应留给 B2：

```text
B2 = B1 + 最低成交量 / 最低持仓量 / 报价宽度过滤
```

## 8. 缺失值和异常值处理

B1 对成交量和持仓量使用保守处理：

```text
volume 缺失 -> 0
open_interest 缺失 -> 0
volume < 0 -> 0
open_interest < 0 -> 0
```

如果所有候选合约成交量和持仓量都为 0，则排序自然退化为：

```text
delta_dist 升序
option_code 升序
```

这与 B0 的结果接近，避免因为数据缺失引入随机性。

## 9. 防未来函数口径

B1 使用 T 日收盘后已经可见的成交量和持仓量，为 T+1 日开仓排序。

允许使用：

```text
T 日期权链的 volume
T 日期权链的 open_interest
T 日收盘后计算出的 Delta 和 moneyness
T 日手续费和保证金参数
```

不允许使用：

```text
T+1 日成交量
T+1 日持仓量
T+1 日开盘后价格波动
持仓后才知道的止损路径
未来到期是否赚钱
```

因此，B1 不引入未来信息。

## 10. B1 预期影响

从期权交易逻辑看，B1 可能带来以下变化：

- 更少选到深虚、低成交、偶发报价跳变的合约。
- 止损中由低流动性异常价触发的比例可能下降。
- ETF、股指和主流商品品种的进入优先级可能提高。
- 单侧 P/C 偏离可能有所缓和，但不会被规则主动控制。
- 收益可能上升，也可能下降；因为更活跃合约不一定权利金最厚。
- 回撤可能下降，但如果主要亏损来自真实行情跳变，B1 不一定能明显改善。

这一版不能用“收益更高”作为唯一判断标准，还应同步看：

- 平均成交量。
- 平均持仓量。
- 开仓合约中 volume=0 的比例。
- 开仓合约中 OI=0 的比例。
- 止损合约的成交量和持仓量分布。
- ETF/股指/商品的开仓占比变化。
- 同一品种内 Top5 合约是否更集中在可交易执行价。

## 11. 与 B0 对比时必须固定的内容

B1 与 B0 对比时，必须固定：

- 回测区间。
- 初始资金。
- 品种池。
- 费用表。
- 保证金率表。
- 滑点参数。
- 止损倍数。
- 成交量限制。
- 次近到期口径。
- Delta 上限。
- 每侧最多 5 个合约。
- 总保证金 50%。

唯一允许变化：

```text
品种扫描排序 + 合约 Top5 排序
```

如果后续结果变化很大，应优先排查：

- 初期开仓是否只因排序发生变化。
- 是否有品种在总保证金边界处被挤出或提前进入。
- P/C 结构是否因更活跃一侧候选更多而自然偏移。
- 是否减少了低成交合约带来的异常止损。

## 12. 一句话定义

B1 是在标准 B0 上做的最小可交易性修正：

```text
仍然全品种、低 Delta、次近到期、双侧卖权、50% 保证金、2.5x 止损；
但品种和合约不再按代码顺序或单纯 Delta 接近度优先，
而是优先选择当日成交量和持仓量更高的品种与合约。
```
