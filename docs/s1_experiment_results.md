# S1 实验结果记录

本文档用于记录 S1 卖权策略的结构实验结果。所有实验默认以
[s1_strategy_improvement_framework.md](s1_strategy_improvement_framework.md)
为设计约束：S1 是低 Delta 卖权、收取 theta/vega carry、通过组合层 stress
budget 控尾部风险的主 carry 引擎。

## 固定实验协议

- 样本区间：`2025-03-01` 到 `2025-06-30`。
- 品种范围：默认全品种池，除非实验说明里特别注明。
- 策略范围：只测 S1，关闭 S3/S4。
- 核心目标：年化贡献 `3.5%` 到 `5.0%`，最大回撤 `1.2%` 到 `1.8%`，Sharpe 至少 `1.2`，Calmar 至少 `2`。
- 主要归因检查：收益应主要来自 theta 和 vega carry，而不是异常 delta 暴露、异常报价或未来函数。
- 提高仓位的实验必须同时报告平均/最大保证金使用率、平均/最大 stress 使用率、最大单日亏损和止损次数。
- 每轮只改变一个主要逻辑模块，避免把入场、选腿、仓位、退出和组合约束混在一起。

## 当前阶段判断

候选评分修改已经阶段性成功。`s1_risk_reward_v1` 将合约选择从单纯 target
delta 改为 fee-adjusted premium / stress_loss、theta / stress_loss、premium /
margin、流动性和 gamma/vega 惩罚的综合排序，在固定区间里显著改善了回撤、止损和收益形态。

但 `s1_risk_reward_v2_budget` 证明不能直接粗暴提高预算。提高 stress budget 后，
4 月尾部风险重新暴露，收益和回撤都明显恶化。因此后续实验应先验证入场边际优势，再考虑动态提高预算。

2026-04-24 盘点后补充：E1 应视为反向消融实验，而不是主线升级实验。当前 repo 中完整
falling/carry 入场框架已经存在，主要体现在 `config_s1_only_stop25_noprotect_volregime.json`；
V1 则是风险收益评分基准，但未启用 `s1_falling_framework_enabled`。下一轮先仍以 V1 为基准，
只把硬止损从 `1.8x` 调整到 `2.5x`，验证止损倍数单变量影响。

## 已完成实验归因

### Baseline

- 主要问题集中在 4 月极端波动窗口。
- 止损次数过多，卖方暴露没有被 premium/stress 质量过滤。
- 最差品种主要包括 `SC`、`PG`、`TA`。

### V1

- 评分改动显著降低回撤和止损，Calmar 达标。
- 收益中枢仍不足，年化只有 `1.70%`，距离 S1 目标贡献 `3.5%` 到 `5.0%` 还有差距。
- 入场质量仍有问题：部分开仓存在 IV/RV spread 为负、RV 抬升、合约 IV 上行。
- Greek 归因里 theta 为正，但 vega 仍为负，说明“卖波动率赚钱”的链条还没有完全打通。

### V2

- 直接提高预算后，4 月尾部亏损重新放大。
- 这不是 V1 逻辑的自然放大版，而是把不够优质的入场也一起放大。
- 后续禁止先做粗暴加仓，应先验证入场边际优势和波动率状态。

### Stop25

- 本轮只改一个变量：`premium_stop_multiple` 从 `1.8` 调整到 `2.5`。
- 配置复核：除 `strategy_version` 外，有效参数只有 `premium_stop_multiple` 不同。
- 代码复核：在当前 commit 下重跑 V1，NAV 与 diagnostics 和旧 V1 完全一致；orders 仅存在 `entry_iv_trend` 约 `1e-16` 量级浮点尾差。
- 路径复核：`2025-03-10` 和 `2025-03-11` 的初期开仓完全一致；第一次分叉来自 V1 在 `2025-03-11 09:07` 对 TA 触发 `1.8x` 止损，而 Stop25 继续持仓到 `2025-03-12 14:16` 才以更大亏损止损。
- 结果不通过。止损次数从 V1 的 `44` 降到 `28`，但总收益从 `+0.85%` 变成 `-2.22%`，最大回撤从 `-0.46%` 放大到 `-2.81%`。
- 4 月是核心损失窗口，单月 PnL 约 `-26.18万`，最差单日 `-1.87%`。
- 最差品种集中在 `PG`、`TA`、`MO`、`SH`，其中 `PG` 单品种 PnL 约 `-17.61万`，单次止损约 `-18.09万`。
- Greek 归因恶化：theta 仍为正约 `+26.23万`，但 vega 亏损扩大到约 `-29.20万`，delta 也转为约 `-6.09万`。
- 结论：在当前 V1 选仓条件下，放宽硬止损不是过滤噪音，而是让坏仓位持有更久、亏损更深。后续不应把 `2.5x` 作为 V1 基准止损，除非先引入更严格的入场质量或异常价格确认。

### F0

- 本轮使用完整 falling/carry 入场框架，文件由 `config_s1_only_stop18_noprotect_volregime.json` 改名为 `config_s1_only_stop25_noprotect_volregime.json`；内容本身已经是 `2.5x` 止损。
- 入场质量确实明显变干净：`open_sell=163` 中，负 IV/RV carry、IV/RV 升温、合约 IV 上行均为 `0`。
- 结果仍不通过：总收益 `-0.18%`，年化 `-0.37%`，最大回撤 `-0.95%`，止损 `52` 次。
- Greek 归因：theta 约 `+26.31万`，vega 约 `-15.30万`，gamma 约 `-24.76万`，delta 约 `-7.84万`。
- 最差品种为 `AL`、`MA`、`UR`、`PG`、`HO`；说明虽然入场更干净，但旧候选评分仍会选到风险收益比不够好的腿。

### F1

- 本轮只在 F0 上叠加 V1 的 `risk_reward` 候选评分，其余 falling framework、`2.5x` 止损、noprotect、动态预算和组合风控尽量不动。
- 相对 F0 有明显改善：总收益从 `-0.18%` 到 `+0.33%`，最大回撤从 `-0.95%` 到 `-0.69%`，止损从 `52` 次降到 `29` 次，开仓卖单从 `163` 降到 `124`。
- 入场质量仍保持干净：负 IV/RV carry、IV/RV 升温、合约 IV 上行均为 `0`。
- Greek 归因较 F0 改善：vega 亏损从约 `-15.30万` 收窄到约 `-11.40万`，theta 约 `+25.62万`，但 delta 亏损扩大到约 `-11.27万`。
- 结论：完整 falling framework 加 risk_reward 是方向正确的合成版本，但在当前参数和结构下还不足以成为主线。它比 F0 更优，但仍低于 V1 的收益/Calmar，也没有达到 S1 年化 `3.5%` 到 `5.0%` 的目标。

### F1 结构拆解

- 动作归因：`29` 次 `sl_s1` 合计亏损约 `-9.90万`，`67` 次提前换月合计收益约 `+9.70万`，`11` 次到期收益约 `+1.55万`。核心矛盾是止损亏损几乎吃掉了全部换月和到期 carry。
- 月度归因：3 月约 `-0.06%`，4 月约 `-0.59%`，5 月约 `+0.42%`，6 月约 `+0.16%`。4 月 `2025-04-07` 是最大单日亏损，单日约 `-0.53%`。
- 方向归因：认购平仓合计约 `+1.36万`，认沽平仓合计约 `-0.02万`，但止损层面认沽亏约 `-5.73万`、认购亏约 `-4.17万`。认沽不是整体亏损很多，而是尾部止损更重。
- 品种归因：最差 product-side 为 `MA P` 约 `-4.88万`、`UR C` 约 `-1.33万`、`SR C` 约 `-0.93万`、`PG C` 约 `-0.75万`、`HO C` 约 `-0.52万`。最大单日亏损主要来自 `2025-04-07` 的三笔 `MA P` 止损。
- 板块归因：`energy_chem` 合计约 `-4.30万`，`equity_core` 约 `-0.97万`，`softs` 约 `-0.63万`；正贡献主要来自 `SC`、`JD`、油脂油料、黑链、贵金属和部分基本金属。
- 双侧暴露粗拆：按“同日、同品种、同到期同时开 C/P”粗略定义，双侧开仓代码后续合计约 `-2.65万`，非双侧开仓代码约 `+4.00万`。其中双侧认沽约 `-2.36万`，说明机械双卖可能拖累 F1，需要把“条件双卖”和“默认双卖”拆开实验。
- 入场评分有效但不充分：最终止损的认购平均 score 更低、stress 更高，说明认购侧评分能部分识别坏仓；最终止损的认沽平均 score 反而不低，且 `entry_rv_trend` 均值略为正，说明认沽侧还需要更强的 RV 不升温、单腿尾损或品种动量惩罚。
- Greek 归因：theta 约 `+25.62万`，vega 约 `-11.40万`，gamma 约 `-22.70万`，delta 约 `-11.27万`，residual 约 `+24.29万`。F1 仍不是干净的 theta/vega carry，收益被 gamma/delta 尾部和残差口径明显扰动。
- 诊断结论：F1 的入场过滤是有效的，问题已经从“有没有 carry”转为“carry 是否足以覆盖尾部止损和双侧结构风险”。下一步不应先加仓，而应先拆 `条件单侧/条件双卖`、`认沽侧额外 RV/动量过滤`、`MA/energy_chem 集中尾损约束` 和 `noprotect 是否合适`。

### F2

- 本轮以 F1 为基准，只加入方向选择和条件双卖。P/C 顶部候选先按 risk-reward 分数比较，并用 `5` 日标的动量惩罚逆势卖权；只有低波/降波、两边分数接近且标的走势中性时才允许双卖。
- 结果不通过：总收益从 F1 的 `+0.33%` 降到 `+0.12%`，年化从 `+0.66%` 降到 `+0.25%`，Calmar 从 `+0.96` 降到 `+0.38`。最大回撤略降，从 `-0.69%` 到 `-0.64%`，但收益牺牲过多。
- 开仓卖单从 `124` 降到 `110`，说明方向选择确实减少了交易；但止损从 `29` 增加到 `34`，尤其认购止损从 `16` 次增加到 `21` 次。
- 双侧开仓粗拆改善不成立：双侧 group 从 F1 的 `23` 个降到 `10` 个，但双侧开仓代码后续仍亏约 `-3.09万`，非双侧代码仅赚约 `+1.02万`。这说明 F2 的条件双卖仍不够严格，甚至可能把保留下来的双卖集中在更差的时点。
- 最差 product-side 从 F1 的 `MA P -4.88万` 变为 `AL P -2.61万`、`MA P -1.70万`、`UR C -1.33万`、`SR C -1.01万`、`PF C -0.86万`。`MA P` 尾损被削弱，但新的认沽/认购尾损出现，方向选择本身还不能解决尾部。
- Greek 归因：theta 从约 `+25.62万` 降到 `+23.80万`，gamma 亏损从约 `-22.70万` 收窄到 `-19.60万`，delta 亏损从约 `-11.27万` 收窄到 `-5.13万`，vega 约 `-11.05万` 基本未改善。F2 降低了一些方向/gamma 风险，但也削弱了 carry。
- 结论：方向选择这个方向保留，但当前“条件双卖”不应继续作为默认增强。下一步应先做 `F2b_strict_single_side` 作为消融：完全禁止同品种同到期双卖，只保留最佳单侧，然后再叠加 F3 的 Put 尾损过滤。

### F2b

- 本轮只在 F2 上关闭条件双卖，保留方向选择和逆势动量惩罚，目的是隔离“条件双卖”本身的影响。
- F2b 比 F2 改善：总收益从 `+0.12%` 到 `+0.19%`，最大回撤从 `-0.64%` 到 `-0.54%`，最差单日从 `-0.47%` 到 `-0.36%`，Calmar 从 `+0.38` 到 `+0.70`。
- 但 F2b 仍低于 F1：总收益 `+0.19%` vs F1 `+0.33%`，theta 从 F1 约 `+25.62万` 降到约 `+22.93万`，说明严格单侧确实降风险，但同时砍掉了不少 carry。
- 止损次数仍为 `34` 次，高于 F1 的 `29` 次；认购止损约 `-6.52万`，认沽止损约 `-3.79万`。方向选择后 Call 侧尾损变得更突出。
- `MA P` 尾损从 F1 约 `-4.88万` 被显著压下，但 `AL P -2.61万`、`UR/SR/PF/IO/HO/CU/PG C` 等尾损仍存在。说明下一步不能只靠 side selection，必须做 Put-specific tail filter 和板块/品种状态降级。
- 结论：`F2b` 只作为“严格单侧”的消融参考，不应被直接当成长期主线。短样本里严格单侧降低了回撤，但卖方长期收益仍需要在震荡环境下合理收两边权利金。后续 F3 不再沿用 put-only tail filter，而改为趋势/动量置信度驱动的双卖与偏侧卖权。

### F3 修订后的规则草案

- 趋势状态只作为置信度，不作为方向预测。建议用 `5/10/20` 日标的收益、RV trend、是否突破近期波动区间做合成分数，分成 `range_bound`、`uptrend`、`downtrend`、`uncertain`。
- `range_bound` 或 `uncertain` 且 IV/RV carry 合格时，允许双卖，但两侧都必须满足 premium/stress、theta/stress、流动性和 Delta cap；双卖不再仅由 P/C 分数接近决定。
- `uptrend` 时，Put 是顺势强侧，允许正常 Delta cap 和正常预算；Call 是弱侧，只允许更低 Delta cap、更小 stress budget、更高 premium/stress 门槛。若趋势置信度很高，弱侧可以直接跳过。
- `downtrend` 时，Call 是顺势强侧，Put 是弱侧，规则对称处理。
- 弱侧不是“少卖一边”这么粗糙，而是“更虚、更小、更贵才卖”：例如弱侧 Delta cap 从 `0.10` 压到 `0.05-0.07`，弱侧 score 乘以 `0.4-0.7`，弱侧单笔 stress budget 乘以 `0.3-0.6`。
- 评价 F3 时必须按趋势状态拆解：震荡双卖贡献、上行趋势 Put/Call 分别贡献、下行趋势 Put/Call 分别贡献，以及弱侧止损是否下降。
