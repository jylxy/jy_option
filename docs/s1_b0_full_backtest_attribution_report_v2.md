# S1 B0 全周期回测归因与策略复盘（期权专家深度版）

版本：`s1_b0_standard_stop25_allprod_2022_latest`

回测区间：`2022-01-04 ~ 2026-03-31`

报告日期：`2026-04-27`

## 1. 期权专家审议摘要

B0 证明了一件重要的事：**全品种、低 Delta、次近到期、50% 保证金、持有到期、2.5x 权利金止损的朴素卖权框架，确实可以持续收取 theta；但它还不是一个高质量的空波动率策略。** 最核心的证据是，累计 theta 为 `+8,369,672`，但 gamma 为 `-5,993,907`、vega 为 `-4,895,860`，theta 甚至无法完全覆盖 gamma 与 vega 的合计损耗，最终只能依靠 `+3,118,564` 的 residual 和少量 delta 才留下 `+884,179` 的 S1 PnL。

从卖权经济含义看，B0 当前赚的是“粗糙权利金”，不是我们最终想要的“优质波动率风险溢价”。优质卖波应当表现为：theta 稳定为正，降波环境中 vega 尽量为正，gamma 损耗受控，尾部日不吞掉过多月份收益。但 B0 的画像是：theta 很厚，gamma/vega 也很厚，止损频繁，到期贡献很大，部分月份 P/C 极端偏移，收益效率低。

B0 相对目标的结论很明确：年化收益 `1.62%`，低于 `6%` 目标；最大回撤 `-3.66%`，高于 `2%` 目标；Calmar `0.44`，资金效率不足；vega 累计为负，不满足“卖方应在降波中赚钱”的目标；策略主要覆盖商品期权，ETF/股指期权参与很少，和乐得多品种商品卖权画像有一部分相似，但收益效率、降波加仓、P/C 控制和尾部修复能力仍有差距。

代码和口径层面，B0 的费用、保证金、到期虚值归零和 2.5x 止损确认已经比早期版本严谨很多，因此这份结果可以作为后续 B1、B2、tail-risk、流动性排序和 falling release 实验的核心基准。但 Greek 归因仍需谨慎：近到期和虚值期权的 IV 反推会污染 vega/residual，因此“vega 为负”应被视为强风险信号，而不是逐笔精确会计结论。

## 3. 策略定义

B0 是 S1 卖权研究的朴素基准，不是优化版。它的作用是回答：如果整个产品只做系统化低 Delta 卖权，不加 IV/RV、趋势、降波、流动性排序、组合 tail cluster 等智能规则，底层卖权风险溢价到底能贡献多少。

| 规则维度 | B0 设定 |
|---|---|
| 品种池 | 当前系统可读到的全品种期权，包括商品、ETF、股指，但实际成交主要集中在商品 |
| 到期选择 | 次近到期，`s1_expiry_mode=nth_expiry`，`s1_expiry_rank=2` |
| 选腿 | 只卖虚值期权，`abs(delta) <= 0.10` |
| 合约宽度 | 每个品种每侧最多 5 个合约 |
| P/C | 默认双侧都卖，品种内 Put/Call 预算各半 |
| 仓位 | 总保证金目标 50%，S1 自身上限 50% |
| 品种分配 | 等权，不考虑流动性排序 |
| 退出 | 不止盈，持有到期 |
| 止损 | 权利金达到开仓价 2.5x 后触发，并使用盘中确认逻辑防跳变 |
| 到期 | 虚值归零，实值按内在价值结算 |
| 成本 | 使用真实手续费表、真实保证金率、滑点和成交量限制 |

B0 明确不包含：IV 预热、IV/RV carry、降波确认、forward vega、趋势 P/C 偏移、板块/相关性预算、cash Greek 预算、tail cluster、保护腿、止盈、止损后冷却重开。

## 9. 图表深度解读

### 图 1：净值与回撤

![图1 净值与回撤](../output/analysis_s1_b0_standard_stop25_allprod_2022_latest/01_nav_drawdown.png)

**怎么看：** 上半部分看净值斜率和修复速度，下半部分看回撤的深度、持续时间和是否由少数事件主导。卖权策略不怕小回撤，怕的是回撤来自同一种风险反复发生。

**图上读数：** 全周期最终 NAV `10,805,100`，累计收益 `8.05%`，年化 `1.62%`，最大回撤 `-3.66%`，最大回撤日为 `2022-07-16`。净值峰值出现在 `2026-02-27`，为 `11,073,589`，期末仍有 `-2.42%` 回撤没有修复。

**期权专家判断：** 曲线说明 B0 有 carry，但 carry 斜率太薄。回撤多次由升波、趋势和止损簇集打断，说明“低 Delta + 分散品种”不能自动消除短 gamma/short vega 的尾部风险。

**风险或口径疑点：** 2025-04-07 和 2026-03 的回撤需要结合分钟止损成交价、异常报价过滤和当日期货/标的走势复核。若某些深虚值合约出现瞬时跳价，单日回撤可能被夸大；但即使剔除个别异常，gamma/vega 长期为负的问题仍然存在。

**下一步验证：** 对所有回撤超过 `0.5%` 的日期生成“产品-方向-合约”尾部贡献表，区分真实趋势亏损、IV spike、异常报价止损和到期结算亏损。

### 图 2：保证金、持仓与活跃度

![图2 保证金与持仓](../output/analysis_s1_b0_standard_stop25_allprod_2022_latest/02_margin_positions.png)

**怎么看：** 这张图判断策略是否真的用了仓位，以及仓位是否分散到足够多品种和合约。若保证金低，收益低可能是仓位不足；若保证金高但收益低，问题通常是单位风险收益质量。

**图上读数：** 平均保证金使用率 `47.51%`，最高 `52.15%`。平均活跃卖方品种 `18.34` 个，平均活跃合约 `178.11` 个，平均活跃手数 `1,267.16` 手。平均每品种合约数约 `10.16`，平均每合约手数约 `7.63`。

**期权专家判断：** B0 不是低仓位策略。它已经相当接近 50% 保证金目标，但收益仍只有年化 `1.62%`，说明瓶颈不是“仓位太小”，而是“同样仓位下收来的 theta 质量不够高”。

**风险或口径疑点：** 保证金口径是盯市保证金除以 NAV，不等于券商结算单的精确占用，也不等于初始本金占用。后续如果更严格接入交易所保证金参数，保证金使用率和可开手数可能变化。

**下一步验证：** 计算每个交易日的 `theta / margin`、`premium / margin`、`theta / stress_loss`，并按产品和方向排序，确认哪些品种只是占用仓位而没有贡献有效 carry。

### 图 3：Greeks 时序

![图3 Greeks 时序](../output/analysis_s1_b0_standard_stop25_allprod_2022_latest/03_greeks_timeseries.png)

**怎么看：** 对卖权组合，cash gamma 和 cash vega 通常为负并不奇怪，关键是这些暴露是否在尾部日前已经过高，以及 theta 是否足以补偿。

**图上读数：** 全周期 cash vega 和 cash gamma 长期为负。每日平均 theta PnL 约 `+6,889`，平均 vega PnL 约 `-4,030`，平均 gamma PnL 约 `-4,933`。P/C 极端日期会伴随 delta 方向化，例如 2026-03 的 P/C 均值高达 `28.36`。

**期权专家判断：** B0 不是 delta 中性的精细卖波组合，而是一个低 Delta、多合约、自然偏移的卖方池。平时 theta 能覆盖小波动，但在单边行情或升波中，cash gamma 和 cash vega 会一起打击组合。

**风险或口径疑点：** Greeks 基于模型和 IV 反推，近到期、深虚值、低流动性合约可能导致 vega/gamma 时序放大或失真。因此报告不能只用 Greeks 下结论，必须和实际 PnL、止损和标的价格路径交叉验证。

**下一步验证：** 在尾部日前 5 个交易日做预警回放，检查 cash gamma、cash vega、P/C、short open premium、stress loss 是否已经出现可识别异常。

### 图 4：PnL 归因

![图4 PnL 归因](../output/analysis_s1_b0_standard_stop25_allprod_2022_latest/04_pnl_attribution.png)

**怎么看：** 这张图要回答策略到底赚什么。卖权策略如果只赚 theta、不赚 vega，仍可能成立；但如果 theta 被 gamma 和 vega 大量吞掉，收益效率就会很低。

**图上读数：** 累计 theta `+8,369,672`，累计 vega `-4,895,860`，累计 gamma `-5,993,907`，累计 delta `+285,710`，累计 residual `+3,118,564`，最终 S1 PnL `+884,179`。

**期权专家判断：** 这是本报告最关键的一张图。B0 的 theta 足够厚，但不是“高质量 theta”。如果一个卖波策略长期 vega 为负，说明它没有系统性吃到降波，或者 IV 估算和选腿路径让归因呈现为负 vega。两种情况都需要修。

**风险或口径疑点：** residual 大于最终 PnL 的 3.5 倍，说明离散重估、skew、vanna、volga、成交和模型误差对结果解释很重要。不能把 residual 直接当 alpha，也不能忽视它对净值的支撑。

**下一步验证：** 对每笔 closed lot 做 forward vega 拆解，按开仓 IV、平仓 IV、DTE、option_type、moneyness 和 close_reason 分层，找出 vega 为负到底来自真实升波还是 IV 口径噪声。

### 图 7：波动状态暴露

## 10. 与乐得画像的差距

B0 和乐得有相似之处：都是商品期权为主、低 Delta、以卖权利金为核心、尽量不做复杂盘中 Delta/Gamma 对冲。但差距也很清楚。

| 维度 | B0 表现 | 与乐得画像差距 |
|---|---|---|
| 收益效率 | 年化 1.62%，50% 保证金 | 明显偏低 |
| 回撤 | 最大回撤 -3.66% | 回撤可接受但收益太低，Calmar 不足 |
| Vega | 累计 -489.6 万 | 没有体现降波赚钱 |
| Theta | 累计 +837.0 万 | carry 存在，但质量不足 |
| P/C | 均值 2.28，极端可到 57.73 | 阶段性过度偏 Put |
| 品种 | 实际集中商品，金融期权很少 | 商品画像相似，但缺 ETF/股指容量 |
| 降波加仓 | falling 暴露极低 | 没有吃到最赚钱阶段 |
| 止损 | 2.5x 硬止损，簇集明显 | 需要冷却、重开和异常价过滤 |
| 颗粒度 | 多品种多合约，但头部集中 | 需要 tail cluster 和产品方向预算 |

最重要的差距不是“B0 规则不够复杂”，而是：**B0 没有识别什么时候权利金值得收，也没有在确认降波时提高预算，更没有在尾部簇集时主动降低同类风险。** 后续策略应该沿着“权利金质量 + 降波预算释放 + 尾部预算约束”的方向改，而不是简单加参数。

## 11. 结论与下一步实验

B0 可以保留为正式基准。它证明低 Delta 全品种卖权有底层 carry，也证明 50% 保证金不是收益不足的主要原因。真正需要解决的是：theta 质量、vega 质量、gamma 尾部、P/C 偏移、止损簇集和真实分散度。

建议下一步按以下顺序推进：

1. **B1 流动性/OI 排序继续跑完并对照 B0。** 当前 B1 阶段性净值更好，但 vega/gamma 更差，必须等全周期和尾部月份验证后再判断。
2. **新增 T0 尾部风险诊断。** 对 B0/B1 输出最差 1%/5% 日的产品-方向贡献、ES、tail cluster 和下尾相依。
3. **重做 vega 质量拆解。** 按 DTE、moneyness、close_reason、entry_iv、exit_iv 分层，确认负 vega 来自真实升波还是 IV 口径。
4. **构建 product-side 风险预算。** A-P、A-C、AL-P、AG-C 这类产品方向要有 margin、stress、stop cluster 和 tail contribution 上限。
5. **优化 falling release。** 不是全组合机械放大，而是单品种确认降波、forward vega 干净、tail contribution 低时释放预算。
6. **补 ETF/股指期权诊断。** 研究为什么金融期权实际进入很少，是费用、保证金、合约筛选、数据映射还是流动性约束导致。
7. **升级报告分析包。** 在现有 11 张图之外增加 `tail_product_side_contribution`、`vega_quality_by_bucket`、`stop_slippage_distribution`、`pc_funnel` 四张图。

总体判断：B0 是一个合格的研究基准，但不是可直接上线的 S1。它提供了底座，也暴露了问题。下一版策略的重点不是更复杂，而是更像真正的期权卖方：知道什么时候该收权利金，知道什么时候这份权利金不够补偿尾部风险，也知道在确认降波时如何更大胆但不失控地吃更多预算。
