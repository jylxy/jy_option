# S1 卖权策略研究开篇报告

## 1. 摘要

本项目研究的是一类以“卖出期权、收取权利金、承担受控波动风险”为核心的日频卖权策略。策略目标不是预测单个品种涨跌，而是在中国场内期权市场中系统性承接波动率风险溢价，通过多品种、多执行价、低 Delta、小单腿、纪律化止损和组合风险管理，形成可解释、可复盘、可逐步接近实盘的收益来源。

我们希望最终构建的不是一个短期参数优化模型，而是一套长期卖方策略研究框架。该框架需要回答三个核心问题：

- 市场原始卖权溢价到底有多厚。
- 什么样的过滤、择时和风险预算规则能在不显著过拟合的前提下改善收益回撤比。
- 我们的组合画像能否逐步接近成熟卖方管理人的多品种、多合约、分散收权利金特征。

为了避免一开始就陷入复杂规则和参数优化，我们先建立一个朴素、干净、可复现的 B0 基准。B0 不做 IV/RV、降波、趋势、板块、相关性或 stress score 等智能判断，只在全品种中卖出低 Delta、次近到期的期权，并将保证金目标固定在 50%。后续所有 S1 改进都必须相对 B0 证明增量价值。

## 2. 为什么要研究卖权策略

期权卖方的本质，是向市场提供保险和波动率承接能力。买方支付权利金，获得非线性收益、尾部保护或方向杠杆；卖方收取权利金，承担标的价格大幅波动、隐含波动率上升、流动性恶化和保证金扩张带来的风险。

卖权策略值得研究，并不是因为它天然安全，也不是因为它可以无风险稳定赚钱，而是因为在很多市场和很多时间段里，期权隐含波动率通常包含风险溢价。市场参与者愿意为保险、杠杆、对冲和确定性支付额外成本，卖方如果能分散承保、控制尾部、控制保证金，就有可能长期收取这部分风险补偿。

因此，卖权策略的核心问题不是：

```text
标的明天涨还是跌？
```

而是：

```text
当前收到的权利金，是否足以补偿未来可能发生的价格波动、升波、跳空、流动性冲击和保证金压力？
```

从投资属性上看，卖权更接近“风险补偿业务”而不是“方向预测业务”。研究重点应放在承保质量、仓位分散、尾部约束、资金效率和回撤恢复能力上。

## 3. 卖权策略的收益与风险

卖权的长期收益可能来自以下来源：

- 时间价值衰减，即 theta 收益。
- 隐含波动率高于未来实际波动率所带来的波动率风险溢价。
- 市场对尾部、对冲和流动性支付的保险费。
- 多品种、多执行价、多方向分散后，单个风险事件没有完全同步爆发所带来的组合平滑。
- 到期时大量虚值期权归零，卖方保留权利金。

但这些收益不是免费午餐。卖方同时承担：

- Short gamma 风险：标的越接近执行价，路径亏损可能越快放大。
- Short vega 风险：隐含波动率上升时卖方通常亏损。
- 跳空风险：止损触发价格和真实成交价格可能差很多。
- 流动性风险：深虚合约或压力期报价可能突然变宽、跳价或无法成交。
- 保证金风险：亏损和保证金占用可能在同一时间上升。
- 到期风险：若到期变成实值期权，需要按内在价值结算，商品期权还可能涉及行权后的期货风险。

所以卖权策略必须从研究起点就承认：它是一个“收取小额保险费、承担少数大风险”的策略。我们要研究的是这份风险补偿是否足够，以及如何把少数大风险控制在组合可承受范围内。

## 4. 对标画像与项目目标

我们希望策略最终接近成熟卖方组合的画像，尤其是乐得这类以多品种、多执行价、分散收权利金为特征的组合。一个更成熟的卖权组合通常不是靠单个大仓位赚钱，而是靠大量小仓位长期分散承保。

目标画像包括：

- 单腿 Delta 较低，避免一开始就承担过强方向暴露。
- 覆盖品种较多，减少单品种事件对组合的冲击。
- 执行价较多，避免风险集中在一个短执行价上。
- P/C 可以随市场状态变化，但不能长期变成单边方向押注。
- 不依赖盘中机械 Delta/Gamma 强平，而依靠事前选腿、仓位、止损和组合风险预算。
- 回撤主要来自少数升波、跳空、趋势突破或流动性冲击阶段，并且冲击后能在降波环境中较快恢复仓位。

项目目标暂定为：

```text
长期年化收益目标：约 6%
最大回撤目标：小于 2%
收益来源目标：以 theta 和 vega 为主，而不是长期依赖 delta 方向暴露
组合画像目标：逐步接近乐得式多品种、多执行价、小单腿卖方组合
```

其中，S1 是核心卖权引擎。它的长期任务是提供稳定的权利金收入和波动率风险溢价暴露。后续所有复杂规则，包括 IV/RV、降波识别、趋势动量、板块相关性、ETF/股指单独评分和 Black76 情景压力测试，都必须先证明自己相对于朴素基准有增量价值。

## 5. 为什么必须先建立 B0 基准

在进入复杂策略之前，我们需要一个干净的参照系。这个参照系要尽量少判断、少优化、少过滤，目的不是直接用于实盘交易，而是回答三个基础问题：

```text
如果什么都不判断，只系统性卖低 Delta 次近到期期权，长期结果如何？
原始卖方权利金是否足够厚？
后续复杂规则到底是在创造收益，还是只是在过拟合和压低仓位？
```

如果后续优化策略跑不过 B0，说明复杂规则没有价值。如果 B0 收益很高但回撤很大，说明卖方风险溢价存在，但需要风控。如果 B0 收益和风险都很低，说明市场原始权利金可能没有我们想象中那么厚，需要重新审视收益目标和仓位假设。

因此，B0 是研究地基，不是最终策略。它的价值在于“简单、粗糙、可解释”。只有知道朴素版本怎么赚钱、怎么亏钱、在哪些场景下死亡，我们才知道复杂版本应该救什么。

## 6. B0 标准基准定义

B0 命名为：

```text
B0: 全品种等权次近到期低 Delta 卖权基准
```

它只表达“全品种系统性收权利金”的原始基准，不表达乐得式择时，也不表达我们的最终策略形态。

### 6.1 策略范围

B0 只启用 S1。

不启用：

- S3 比例价差。
- S4 或其他结构。
- 保护腿。
- IV/RV 过滤。
- 降波或 forward vega 过滤。
- 趋势和动量 P/C 偏移。
- 板块集中度和相关性控制。
- stress score 候选评分。
- stress budget sizing。
- 止盈。
- 止损后冷却期和重开规则。

### 6.2 品种池

默认扫描当前系统可读到的全部期权品种，包括商品期权、ETF 期权和股指期权。

品种上市后不需要人工加入白名单。只要当天存在有效期权链、真实标的价格、合约乘数、手续费和保证金参数，就进入每日扫描。实际能否开仓由候选合约是否存在、整数手数、保证金约束、成交量限制和费用过滤共同决定。

### 6.3 到期选择

B0 使用“次近到期”，代码口径为：

```text
s1_expiry_mode = nth_expiry
s1_expiry_rank = 2
```

这里的“次近到期”不是自然月份的“下个月”，而是当天有效到期序列中按 DTE 从小到大排序后的第 2 个到期日。这个口径更适合商品期权，因为商品期权合约月份并不总是严格对应自然次月。如果当天只有 1 个有效到期日，则该品种当天不开仓。

### 6.4 选腿规则

每天对每个品种的次近到期同时扫描 Put 和 Call：

- Put：只卖 OTM Put。
- Call：只卖 OTM Call。
- Delta：`abs(delta) <= 0.10`。
- Delta 下限：不设硬下限。
- 排序：优先选择最接近 `abs(delta)=0.10` 的合约，再用流动性和合约代码做稳定排序。
- 每个品种、每个方向、每个扫描日最多选择 5 个卖方合约。
- 不买保护腿，不做价差，只记录裸卖基准。

这版 B0 不再“所有符合条件的深虚值合约都卖”。原因是全卖会把品种结构变成交易所挂牌密度测试，而不是策略基准；每侧 5 个低 Delta 合约更接近可解释、可执行、可复盘的执行价梯队。

### 6.5 P/C 结构

B0 不做趋势判断，也不主动偏 Put 或偏 Call。每个品种每天默认尝试双侧卖权：

```text
每个品种预算 = 当日可用 S1 预算 / 当日可交易品种数
Put 侧预算 = 每个品种预算 / 2
Call 侧预算 = 每个品种预算 / 2
```

实际结果中 P/C 仍可能偏离 1:1，这是正常现象，主要来自：

- 某一侧没有满足 `abs(delta)<=0.10` 且 OTM 的合约。
- 某一侧权利金不足以覆盖往返手续费。
- 整数手数和保证金不同。
- 该侧触发止损、到期或成交量限制。
- 某些 ETF、股指、商品品种在某段时间的数据可用性不同。

因此，B0 的目标不是强制 P/C 手数完全相等，而是保证开仓逻辑本身不主观偏向某一侧。

### 6.6 仓位和资金分配

核心仓位参数：

```text
capital = 10,000,000
margin_cap = 0.50
s1_margin_cap = 0.50
margin_per = 0.02
s1_baseline_equal_weight_products = true
s1_baseline_equal_weight_contracts = true
s1_baseline_max_contracts_per_side = 5
```

解释：

- 组合总保证金使用率最高 50%。
- S1 自身保证金使用率最高 50%。
- 每日按当日可交易品种数等权分配预算。
- 每个品种内部 P/C 各拿一半预算。
- 每侧最多 5 个合约，合约之间按剩余预算等权切分。
- 同一品种、同一方向、同一到期允许在后续交易日继续加仓，直到组合和品种预算自然限制住。

实际保证金使用率可能低于 50%。这不一定是 bug，常见原因是合格候选不足、整数手数向下取整、单合约保证金太大、开仓成交量限制、费用过滤或已有仓位浮亏导致 NAV 变化。

### 6.7 成本、成交和流动性

B0 保留实盘必要成本假设：

```text
option_fee_use_broker_table = true
margin_ratio_use_broker_table = true
s1_min_premium_fee_multiple = 1.0
volume_limit_pct = 1.0
execution_slippage_enabled = true
execution_open_slippage_pct = 0.002
execution_close_slippage_pct = 0.002
execution_stop_slippage_pct = 0.005
execution_slippage_apply_to_expiry = false
```

其中 `s1_min_premium_fee_multiple=1.0` 表示单张合约权利金现金额至少要覆盖一开一平的往返手续费。否则即使 Delta 合格，也不值得做。

成交仍使用当前系统里的 T+1 全天 VWAP 口径，并保留成交量限制。这个设定不是为了让 B0 变聪明，而是为了避免用不可成交价格构造收益。

### 6.8 退出规则

B0 默认持有到期，不止盈、不主动滚仓、不做临近到期提前处理。

唯一提前退出是权利金硬止损：

```text
premium_stop_multiple = 2.5
premium_stop_requires_daily_iv_non_decrease = false
intraday_stop_confirmation_enabled = true
```

也就是说，期权价格达到开仓权利金的 2.5 倍时触发止损，但仍使用当前盘中止损防跳变确认逻辑，避免因为深虚值小成交量合约的瞬时异常报价直接打出假止损。

到期处理按真实到期内在价值结算：

- 虚值到期：期权价值归零，卖方保留权利金。
- 实值到期：按内在价值结算卖方亏损。

## 7. B0 预期优点与预期问题

B0 的优点是：

- 足够简单，便于解释。
- 能衡量原始低 Delta 卖权溢价。
- 全品种等权能避免一开始就引入主观品种选择。
- 次近到期通常比临近到期有更平衡的 theta 和 gamma。
- 持有到期能观察原始卖方生命周期，不被频繁止盈和滚仓干扰。
- 50% 保证金目标能测试卖方策略在较高仓位下的真实尾部。

B0 也很可能暴露大量问题：

- 50% 保证金对裸卖策略可能偏激进。
- 全品种等权会把资金分给流动性差、历史短、报价质量差的品种。
- 只看 Delta 会忽略 IV 便宜、RV 抬升、skew steepen 和事件风险。
- 次近到期持有到期仍会暴露临近到期 gamma 风险。
- 不止盈会让已经赚到大部分权利金的仓位继续承担尾部。
- 2.5x 止损不是最大亏损上限，跳空时可能远超 2.5x 成交。
- 深虚 cheap OTM 合约可能出现价格跳变和流动性断层。
- ETF、股指和商品的风险收益特征不同，等权混合可能不合理。
- 长期 P/C 无脑双卖可能在趋势行情中承受明显单侧亏损。

这些问题不是基准失败，而是基准应该暴露的东西。B0 的研究价值，正是帮助我们识别朴素卖权策略的收益厚度和主要死亡方式。

## 8. B0 回测必须输出的结果

B0 回测报告至少输出：

- 总收益、年化收益、年化波动、Sharpe、Calmar、最大回撤。
- 平均保证金使用率、最高保证金使用率、保证金接近 50% 的天数比例。
- 年度、季度、月份收益拆解。
- 最差单日、最差连续 5 日、最大回撤区间。
- 开仓数量、止损数量、到期数量、平均持有天数。
- 按品种、市场、板块、P/C、到期月份拆解 PnL。
- 按 action 拆解 PnL，包括止损、到期、正常持有。
- 到期实值次数、到期实值亏损、虚值归零收益。
- Greek 归因，包括 delta、gamma、theta、vega、residual。
- 手续费、滑点、执行偏离。
- 止损时真实成交价相对 2.5x 阈值的偏离。
- 单品种最大亏损、单合约最大亏损、单日最大亏损来源。

额外建议输出：

- ETF、股指、商品三类市场分别绩效。
- 不同 Delta bucket 的绩效。
- 不同 DTE bucket 的绩效。
- 2025 年 4 月、7 月等关键升波月份的表现。
- 与乐得画像指标的差距，包括活跃品种数、活跃合约数、单合约平均手数、P/C 比例、权利金密度和保证金使用率。

## 9. 如何解释 B0 结果

后续任何版本都必须回答：

```text
是否提高了年化收益？
是否降低了最大回撤？
是否改善了 Calmar？
是否改善了 Vega PnL？
是否减少了 cheap OTM 跳变尾损？
是否提高了单位保证金收益？
是否提高了单位 tail risk 收益？
是否更接近乐得画像？
```

如果某个复杂版本只是降低了仓位、降低了回撤，但收益也消失，它不是有效升级。如果某个复杂版本提高了收益，但靠单品种、单方向或单个时期承担更大尾部，它也不是有效升级。

B0 的意义，是让我们后续每一步都能明确区分：

```text
这是卖权本身的风险溢价，
还是我们规则创造的增量价值，
还是只是参数和样本的偶然性。
```

## 10. 后续研究路线

在 B0 跑通并完成拆解后，S1 的后续研究建议按以下顺序推进：

1. 先确认数据和回测口径：真实标的价格、合约乘数、手续费、保证金、成交量限制、到期结算、异常报价过滤。
2. 再确认 B0 的收益来源：theta、vega、delta、gamma 和 residual 是否符合卖权策略常识。
3. 然后加入最小必要过滤：费用过滤、异常报价过滤、流动性下限、真实可成交约束。
4. 再研究 IV/RV carry：只在隐含波动率相对未来风险有补偿时卖。
5. 再研究降波环境：在确认降波、forward vega 更干净时提高单品种风险预算。
6. 再研究 VCP 和长期低波品种：低波不是天然安全，必须判断是否存在突破前的波动率收缩陷阱。
7. 再研究组合层约束：板块、相关性、cash Greeks、单品种尾损和保证金挤兑。
8. 最后再研究是否引入轻量模型：优先预测 future RV、bad regime probability 或候选交易 forward PnL，而不是直接预测 IV 点位。

这条路线的核心是：先建立朴素基准，再逐步证明每个复杂模块是否真的改善了策略，而不是让复杂性本身掩盖问题。

## 12. 阶段性结论

本阶段最重要的工作不是证明策略已经成功，而是把研究问题定义清楚。S1 卖权策略的研究应从以下共识出发：

- 卖权不是无风险套利，而是系统性承接波动率和尾部风险。
- 收益目标必须同时约束回撤、Vega PnL、保证金使用率和组合画像。
- 对标乐得不是简单复制持仓，而是学习其多品种、多执行价、小单腿、纪律化止损和冲击后恢复仓位的组合特征。
- B0 是所有后续优化的地基。没有 B0，复杂版本的好坏很容易被参数、样本和仓位差异误导。
