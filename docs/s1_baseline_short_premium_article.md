# S1 卖权基准策略说明

## 1. 为什么要做卖权

期权卖方的本质，是向市场提供保险和波动率承接能力。买方支付权利金，获得非线性收益、尾部保护或方向杠杆；卖方收取权利金，承担标的价格大幅波动、隐含波动率上升、流动性恶化和保证金扩张带来的风险。

卖权策略之所以值得研究，不是因为它天然安全，也不是因为它可以稳定无风险赚钱，而是因为在很多市场和很多时间段里，期权隐含波动率通常包含风险溢价。市场参与者愿意为保险、杠杆、对冲和确定性支付额外成本，卖方如果能分散承保、控制尾部、控制保证金，就有可能长期收取这部分风险补偿。

因此，卖权策略的核心不是预测标的涨跌，而是回答一个更基础的问题：

```text
当前收到的权利金，是否足以补偿未来可能发生的价格波动、升波、跳空、流动性冲击和保证金压力？
```

在最朴素的版本里，我们先不试图判断波动贵不贵，也不试图判断趋势、板块、相关性或降波窗口。我们先建立一个“无脑卖低 Delta 次月期权”的基准，用它衡量中国场内期权市场原始卖方溢价到底有多厚。

## 2. 卖权的收益来源

卖权的长期收益可能来自几类来源：

- 时间价值衰减，也就是 theta 收益。
- 隐含波动率高于未来实际波动率所带来的波动率风险溢价。
- 市场对尾部、对冲和流动性支付的保险费。
- 多品种、多执行价、多方向分散后，单个风险事件没有完全同步爆发所带来的组合平滑。
- 到期时大量虚值期权归零，卖方保留权利金。

但这些收益不是免费午餐。卖方同时承担：

- Short gamma 风险，标的价格越接近执行价，路径亏损可能越快放大。
- Short vega 风险，隐含波动率上升时卖方通常亏损。
- 跳空风险，止损触发价格和真实成交价格可能差很多。
- 流动性风险，深虚合约或压力期报价可能突然变宽或跳价。
- 保证金风险，亏损和保证金占用可能在同一时间上升。
- 到期风险，若到期变成实值期权，需要按内在价值结算，商品期权还可能涉及行权后的期货风险。

所以，卖权的研究必须从一开始就承认：它是一个“收取小额保险费、承担少数大风险”的策略。我们要研究的是这种风险补偿是否足够，而不是把高胜率误认为低风险。

## 3. 卖权策略的典型画像

一个成熟的卖权组合通常不是靠单个大仓位赚钱，而是靠大量小仓位分散收取权利金。它一般具有以下特征：

- 单腿 Delta 较低，避免一开始就承担太强方向暴露。
- 品种较多，减少单品种事件对组合的影响。
- 执行价较多，避免风险集中在一个短执行价上。
- P/C 会随市场状态变化，但不能长期变成单边方向押注。
- 不依赖盘中机械 Delta/Gamma 强平，而是依靠事前选腿、仓位、止损和组合风险预算。
- 回撤主要来自少数升波、跳空、趋势突破或流动性冲击阶段。

我们后续希望逐步接近乐得这类卖方组合的画像：多品种、多执行价、小单腿、低 Delta、权利金分散、止损纪律明确，并在冲击后进入降波阶段时能够较快恢复仓位。

## 4. 我们的目标

我们的最终目标不是做一个“看起来收益最高”的回测，而是做一个可以解释、可以复盘、可以逐步接近实盘的卖权系统。

组合目标暂定为：

```text
长期年化收益目标：约 6%
最大回撤目标：小于 2%
收益来源目标：以 theta 和 vega 为主，而不是长期依赖 delta 方向暴露
组合画像目标：尽量接近乐得的多品种、多执行价、小单腿卖方组合
```

其中，S1 是核心卖权引擎。它的长期任务是提供稳定的权利金收入和波动率风险溢价暴露。后续所有复杂规则，包括 IV/RV、VCOS、相关性、板块、ETF/股指单独评分、Black76 情景压力测试，都必须先证明自己相对于朴素基准有增量价值。

## 5. 为什么要先建立朴素基准

在进入复杂策略之前，我们需要一个干净的参照系。这个参照系要尽量少判断、少优化、少过滤，目的不是实盘交易，而是回答三个基础问题：

```text
如果什么都不判断，只系统性卖低 Delta 次月期权，长期结果如何？
原始卖方权利金是否足够厚？
后续复杂规则到底是在创造收益，还是只是在过拟合和压低仓位？
```

如果后续优化策略跑不过这个基准，说明复杂规则没有价值。如果基准收益很高但回撤很大，说明卖方风险溢价存在，但需要风控。如果基准收益和风险都很低，说明市场原始权利金可能没有我们想象中那么厚，需要重新审视收益目标和仓位假设。

因此，这个基准是研究的地基，不是最终策略。

## 6. B0 基准策略定义

这个基准命名为：

```text
B0: 全品种等权次月低 Delta 卖权基准
```

### 6.1 策略范围

策略只使用 S1。

不启用：

- S3 比例价差。
- S4 或其他结构。
- 保护腿。
- IV/RV 过滤。
- VCOS 降波评分。
- 趋势和动量 P/C 偏移。
- 板块和相关性主动优化。
- 流动性加权资金分配。
- 止盈。
- 主动滚仓。

### 6.2 品种池

默认使用当前系统可覆盖的全品种，包括：

- 商品期权。
- ETF 期权。
- 股指期权。

品种进入样本的前提是本地数据中该品种已经上市并有可用合约数据。短历史品种不提前剔除，但正式报告中需要单独标记上市时间和有效样本长度。

### 6.3 合约期限

只交易次月到期合约。

这里的“买入次月”在本文档中统一表述为“选择次月合约建立 S1 卖方仓位”。S1 的交易方向仍然是卖出期权义务方，而不是买入期权权利方。

不交易：

- 当月合约。
- 远月合约。
- 周期过短的临近到期合约。
- 跨期结构。

### 6.4 行权价和 Delta

只交易绝对 Delta 小于 0.1 的虚值期权。

规则为：

```text
卖 Put: delta < 0 且 abs(delta) < 0.1
卖 Call: delta > 0 且 abs(delta) < 0.1
```

这里的 Delta 指期权合约本身的权利方 Delta，用于筛选虚值程度；实际卖出后，组合持仓 Delta 的符号会反过来。

只要符合次月、虚值、Delta 小于 0.1 的条件，就进入候选，不再根据 IV/RV、趋势、波动状态或收益风险比做主观筛选。

如果同一品种同一方向存在多个符合条件的执行价，基准允许铺开到多个执行价。资金分配按等权原则处理，不按流动性或评分倾斜。

### 6.5 资金和保证金

组合目标保证金使用率固定为 50%。

资金分配原则：

```text
全品种等权
不考虑流动性加权
不考虑波动率高低加权
不考虑品种历史表现加权
不考虑板块或相关性主动倾斜
```

实现上，先把目标保证金预算分配给当日可交易品种，再在品种内部按符合条件的方向和合约等权铺开。若由于合约乘数、最小手数、保证金或可用候选不足导致无法精确达到 50%，则记录实际保证金使用率。

### 6.6 费用和保证金率

费用和保证金率默认跟交易所口径一致。

基准要求：

- 使用当前系统中已经接入的交易所或期货公司费率表。
- 使用交易所保证金率和合约乘数。
- 保证金每日按盯市价格和当前标的价格重算。
- 不使用人为调低的手续费。
- 不使用乐观的保证金折扣。

如果 ETF、股指、商品的费率或保证金规则不同，按各自市场口径分别处理，不混用。

### 6.7 开仓规则

每日扫描全品种。

对每个品种：

```text
1. 找到次月合约。
2. 找到 abs(delta) < 0.1 的虚值 Put 和 Call。
3. 不做 IV/RV、趋势、降波、流动性评分。
4. 按等权资金预算建立卖方仓位。
5. 直到达到该品种预算或组合 50% 目标保证金。
```

开仓时点沿用当前回测系统的日频/分钟执行口径。若使用服务器分钟数据执行，则按当前系统已有的成交与滑点规则记录，不在基准中额外优化执行。

### 6.8 持仓和退出

持有到期。

不止盈。

不主动滚仓。

唯一提前退出规则是权利金止损：

```text
当期权价格上涨到开仓权利金的 2.5 倍时，触发止损。
```

止损仍沿用当前系统的防跳变规则。也就是说，不因为单个异常报价立刻认定必须成交，而是使用当前已有的异常价格和跳价确认逻辑，尽量避免深虚低流动性合约的一次性假报价导致错误止损。

若未触发止损，则持有到期：

- 到期仍为虚值，期权价值归零，卖方保留权利金。
- 到期变为实值，按内在价值结算，亏损计入到期收益。
- 商品期权如涉及行权和期货风险，在回测中按当前系统设定的到期结算口径处理，并在报告中单独标记。

## 7. B0 基准不做什么

为了保持基准干净，B0 明确不做以下事情：

- 不判断 IV 是贵还是便宜。
- 不判断 IV 是否正在下降。
- 不判断 RV 是否正在上升。
- 不区分 falling、low stable、normal、high rising。
- 不根据趋势调整 P/C。
- 不根据板块和相关性调仓。
- 不根据成交量或流动性分配更多资金。
- 不按 `premium/tail_stress` 或 `theta/tail_stress` 排序。
- 不使用 Black76 情景压力测试做 sizing。
- 不做止盈。
- 不做临近到期提前处理。

这些功能都留给后续改进版本。B0 的价值正是“简单、粗糙、可解释”。

## 8. B0 基准的预期优点

B0 的优点是：

- 足够简单，便于解释。
- 能衡量原始低 Delta 卖权溢价。
- 全品种等权能避免一开始就引入主观品种选择。
- 次月合约通常权利金比近到期更厚，且比远月更有 theta 效率。
- 持有到期能观察原始卖方生命周期，不被频繁止盈和滚仓干扰。
- 50% 保证金目标能测试卖方策略在较高仓位下的真实尾部。

这个基准会告诉我们：如果完全不聪明，只系统性卖低 Delta 次月期权，市场本身给不给钱。

## 9. B0 基准的预期问题

B0 很可能会暴露大量问题：

- 50% 保证金对裸卖策略可能偏激进。
- 全品种等权会把资金分给流动性差、历史短、报价质量差的品种。
- 只看 Delta 会忽略 IV 便宜、RV 抬升、skew steepen 和事件风险。
- 次月持有到期会暴露临近到期 gamma 风险。
- 不止盈会让已经赚到大部分权利金的仓位继续承担尾部。
- 2.5x 止损不是最大亏损上限，跳空时可能远超 2.5x 成交。
- 深虚 cheap OTM 合约可能出现价格跳变和流动性断层。
- ETF、股指和商品的风险收益特征不同，等权混合可能不合理。
- 长期 P/C 无脑双卖可能在趋势行情中承受明显单侧亏损。

这些问题不是基准失败，而是基准应该暴露的东西。只有知道朴素版本怎么死，我们才知道复杂版本应该救什么。

## 10. B0 回测必须输出的结果

B0 回测报告至少输出：

- 总收益、年化收益、年化波动、Sharpe、Calmar、最大回撤。
- 平均保证金使用率、最高保证金使用率、保证金达到 50% 的天数比例。
- 年度、季度、月份收益拆解。
- 最差单日、最差连续 5 日、最大回撤区间。
- 开仓数量、止损数量、到期数量、平均持有天数。
- 按品种、市场、板块、P/C、到期月份拆解 PnL。
- 按 action 拆解 PnL，包括止损、到期、正常持有。
- 到期实值次数、到期实值亏损、虚值归零收益。
- Greek 归因，包括 delta、gamma、theta、vega、residual。
- 手续费、滑点、执行偏离。
- 止损时真实成交价相对 2.5x 阈值的偏离。
- 单品种最大亏损、单合约最大亏损、单日最大亏损来源。

额外建议输出：

- ETF/股指/商品三类市场分别的绩效。
- 不同 Delta bucket 的绩效。
- 不同 DTE bucket 的绩效。
- 2025 年 4 月、7 月等关键升波月份的表现。
- 与乐得画像指标的差距，包括活跃品种数、活跃合约数、单合约平均手数、P/C 比例和保证金使用率。

## 11. 如何使用 B0 基准

B0 是后续所有策略升级的共同参照物。

后续任何版本都必须回答：

```text
是否提高了年化收益？
是否降低了最大回撤？
是否改善了 Calmar？
是否改善了 Vega PnL？
是否减少了 cheap OTM 跳变尾损？
是否提高了单位保证金收益？
是否提高了单位 tail risk 收益？
是否更接近乐得画像？
```

如果某个复杂版本只是降低了仓位、降低了回撤，但收益也消失，它不是有效升级。如果某个复杂版本提高了收益，但靠单品种、单方向或单个时期承担更大尾部，它也不是有效升级。

B0 的意义，是让我们后续每一步都能明确区分：

```text
这是卖权本身的风险溢价，
还是我们规则创造的增量价值，
还是只是参数和样本的偶然性。
```

## 12. 下一步

下一步应先实现或配置 B0 回测，而不是继续优化现有复杂 S1。

推荐顺序：

1. 固化 B0 配置文件。
2. 确认全品种池、次月合约识别、Delta 小于 0.1 的候选生成。
3. 确认 50% 目标保证金和等权资金分配口径。
4. 确认交易所费用、保证金率和合约乘数。
5. 确认 2.5x 止损与防跳变逻辑。
6. 跑一个短区间样本，核对订单和持仓是否符合“无脑基准”定义。
7. 再跑 2025 年全年或更长区间，作为后续所有 S1 改进的基准线。
