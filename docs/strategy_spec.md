## 一、策略总览

本策略是以卖出深虚值期权收取权利金为核心的低频组合策略，包含三个子策略：

| 子策略 | 名称 | 角色 | 贡献 |
|--------|------|------|------|
| **S1** | 垂直价差保护卖权 | 主策略（收益引擎） | 提供稳定Theta收入 |
| **S3** | 蝶式比例价差 | 辅助策略（收益增强） | 利用结构性价差增厚收益 |
| **S4** | 尾部对冲 | 保护策略（风控） | 买入深虚值期权对冲极端风险 |

**核心逻辑**: 在20个流动性最好的商品/金融期权品种上，持续卖出深虚值期权（|delta|<0.15），利用时间衰减（Theta）获利，同时通过买入保护腿和尾部对冲控制尾部风险。

**回测表现**:

|------|------|
| 年化收益率 | +26.09% |

| 最大回撤 | -6.84% |
| 夏普比率 | 1.29 |
| 卡玛比率 | 3.81 |
| 月度胜率 | 74% (20/27) |
| 月均盈利 | +2.86% |

## 三、S1 垂直价差保护卖权

### 3.1 策略结构

双卖（Put+Call），每方向独立执行：
- **卖腿**（sell）：卖出深虚值期权，收取权利金
- **保护腿**（buy）：买入更接近平值的同类期权，限制最大亏损

### 3.2 开仓条件

**触发时机**：当某品种存在到期日满足 DTE ≈ 35 天的合约时触发开仓。

具体逻辑：
1. 遍历该品种所有到期日
2. 筛选 15 ≤ DTE ≤ 90 的到期日
3. 选择 DTE 最接近 35 天的那个到期日
4. 在该到期日首次出现时触发开仓

**注意**：同品种不限制同时持有不同到期日的S1仓位（允许重叠）。

### 3.3 卖腿合约选择

对每个方向（Put和Call）独立选择：

**Put卖腿筛选条件**：
- `option_type = 'P'`
- `moneyness < 1.0`（虚值）
- `delta < 0` 且 `|delta| < 0.15`（深虚值）
- `option_close >= 0.5`（最低权利金）

**Call卖腿筛选条件**：
- `option_type = 'C'`
- `moneyness > 1.0`（虚值）
- `delta > 0` 且 `delta < 0.15`（深虚值）
- `option_close >= 0.5`（最低权利金）

**排序选择**：在满足条件的合约中，计算效率值并选择最高的：
```
效率 = option_close × multiplier / 单手保证金
```
选择效率最高的合约作为卖腿。

### 3.4 保护腿合约选择

**Put保护腿**：
- `option_type = 'P'`
- `moneyness < 1.0`
- `delta < 0` 且 `|delta| < 0.25`
- `option_close >= 0.5`
- `strike > sell_strike`（比卖腿更接近平值）
- 选择 `|delta|` 最大的合约

**Call保护腿**：
- `option_type = 'C'`
- `moneyness > 1.0`
- `delta > 0` 且 `delta < 0.25`
- `option_close >= 0.5`
- `strike < sell_strike`（比卖腿更接近平值）
- 选择 `delta` 最大的合约

### 3.5 手数计算

```
每方向手数 = max(1, int(NAV × margin_per / 2 × iv_scale / 单手保证金))
保护腿手数 = max(1, 卖腿手数 // 2)
```

其中：
- `margin_per = 0.02`（每品种占NAV的2%保证金额度）
- 除以2是因为双卖（Put和Call各占一半）
- `iv_scale`：IV反转因子（见3.7节）

### 3.6 保证金计算

单手保证金采用中金所公式：
```
Call: (权利金 + max(S×margin_ratio - OTM额, 0.5×S×margin_ratio)) × 乘数
Put:  (权利金 + max(S×margin_ratio - OTM额, 0.5×K×margin_ratio)) × 乘数

其中 OTM额:
  Call = max(strike - spot, 0)
  Put  = max(spot - strike, 0)
```

### 3.7 IV反转机制

```python
IV分位数 = rolling_percentile(ATM隐含波动率, window=252, min_periods=60)
iv_scale = 0.5 if IV分位数 > 75 else 1.0
```

ATM隐含波动率筛选条件：`moneyness ∈ [0.95, 1.05]` 且 `dte ∈ [15, 90]` 且 `implied_vol > 0`

当IV处于过去一年的高位（>75分位）时，将手数减半，降低高波动率环境下的风险暴露。

### 3.8 IV环境过滤（波动率高位暂停开仓）

```python
if IV分位数 > 80:
    暂停该品种S1/S3新开仓（含止盈重开）
    S4不受影响（买方策略，高波时更有价值）
```

当品种IV处于历史80%分位以上时，完全暂停该品种的卖权新开仓和止盈重开。
已有持仓不受影响（不主动平仓），只是不新增卖权敞口。

回测验证：该规则使夏普从1.13提升到1.47（+30%），回撤从-8.03%降到-7.05%，
年化反而从+20.71%提升到+22.50%。原因是高IV环境下新开的卖权虽然权利金高，
但标的波动也大，亏损概率更高，暂停开仓反而避开了最差的交易。

### 3.9 止盈

- **条件**：`profit_pct >= 40%` 且 `DTE > 5`
- **profit_pct**：`(open_price - cur_price) / open_price`（卖腿）
- **执行**：平仓该卖腿及其关联的保护腿
- **重开**：若 `DTE > 10` 且 IV分位≤80%，立即在同一天重新选择合约开仓（使用完整margin_per，受保证金上限约束）

### 3.10 到期平仓

- **条件**：`DTE <= 1`
- **执行**：平仓该卖腿及其关联保护腿
- 无数据时（到期日无行情），期权价格视为0（到期作废）

## 四、S3 蝶式比例价差

### 4.1 策略结构

双卖（Put+Call），每方向独立执行：
- **买腿**（buy）：1手
- **卖腿**（sell）：3手（比例因子 `s3_ratio = 3`）
- **保护腿**（protect）：3手

结构比例 = 买1 : 卖3 : 保护3

### 4.2 开仓条件

与S1相同的触发时机。**优先级高于S1**：同一品种同一天，先开S3再开S1。

**限制**：同品种同方向（P或C）同一时间只允许一组S3卖腿持仓。

### 4.3 买腿合约选择

**Put买腿**：
- `option_type = 'P'`
- `moneyness < 1.0`
- `delta < 0` 且 `0.10 <= |delta| <= 0.20`
- `option_close >= 0.5`
- 选择 `|delta|` 最接近 0.15 的合约

**Call买腿**：
- `option_type = 'C'`
- `moneyness > 1.0`
- `delta > 0` 且 `0.10 <= delta <= 0.20`
- `option_close >= 0.5`
- 选择 `delta` 最接近 0.15 的合约

### 4.4 卖腿合约选择

**Put卖腿**：
- `option_type = 'P'`
- `moneyness < 1.0`
- `delta < 0` 且 `0.05 <= |delta| <= 0.15`
- `option_close >= 0.5`
- `strike < buy_strike`（比买腿更虚值）
- 选择 `option_close` 最高的合约（权利金最大化）

**Call卖腿**：
- `option_type = 'C'`
- `moneyness > 1.0`
- `delta > 0` 且 `0.05 <= delta <= 0.15`
- `option_close >= 0.5`
- `strike > buy_strike`（比买腿更虚值）
- 选择 `option_close` 最高的合约

### 4.5 保护腿合约选择

**Put保护腿**：
- `option_type = 'P'`
- `moneyness < 1.0`
- `option_close >= 0.1`（保护腿价格门槛更低）
- `strike < sell_strike`（比卖腿更虚值）
- 目标行权价：`target = sell_strike - (spot - sell_strike) × 0.5`
- 选择行权价最接近 target 的合约

**Call保护腿**：
- `option_type = 'C'`
- `moneyness > 1.0`
- `option_close >= 0.1`
- `strike > sell_strike`
- 目标行权价：`target = sell_strike + (sell_strike - spot) × 0.5`
- 选择行权价最接近 target 的合约

### 4.6 手数计算

```
买腿手数 = max(1, int(NAV × margin_per / 2 × iv_scale / (单手卖腿保证金 × s3_ratio)))
卖腿手数 = 买腿手数 × s3_ratio
保护腿手数 = 卖腿手数  （与卖腿相同）
```

### 4.7 止盈

- **条件**：`profit_pct >= 30%` 且 `DTE > 5`
- **执行**：平仓卖腿及其关联买腿、保护腿
- **重开**：若 `DTE > 10` 且 IV分位≤80%，立即重新开仓（受保证金上限约束）

### 4.8 到期平仓

- **条件**：`DTE <= 1`
- 平仓全组（买+卖+保护）

## 五、S4 尾部对冲（v2：全品种+15天持仓限制）

### 5.1 策略结构

纯买入策略（不卖出），作为组合的尾部保护：
- 买入最深虚值Put（保护下行）
- 买入最深虚值Call（保护上行）

### 5.2 参与品种

**全部20品种**（与S1/S3相同的品种池）。

> v1版本仅限4品种（中证1000、沪金、原油、沪铜），v2扩展到全品种。
> S4是买方策略不占保证金，多品种几乎无额外成本，但能捕捉更多波动率扩张机会。

### 5.3 开仓条件

与S1/S3相同的触发时机。**限制**：同品种同到期日只允许一组S4持仓。

### 5.4 合约选择

**Put**：
- `option_type = 'P'`
- `moneyness < 1.0`
- `option_close >= 0.1`
- 选择 `moneyness` 最小的合约（最深虚值）

**Call**：
- `option_type = 'C'`
- `moneyness > 1.0`
- `option_close >= 0.1`
- 选择 `moneyness` 最大的合约（最深虚值）

### 5.5 手数计算

```
每方向预算 = NAV × s4_prem / S4品种数 / 2
每方向手数 = max(1, int(预算 / (option_close × multiplier)))
手数上限 = 5  （严格限制）
```

其中 `s4_prem = 0.005`（NAV的0.5%用于尾部对冲）

### 5.6 平仓条件

- `DTE <= 1` 时平仓（到期）
- **持仓超过15天时平仓**（v2新增，避免Theta加速衰减区间）

> 15天持仓限制是S4 v2的核心改进。深虚值期权的Theta在最后10天加速衰减，
> 15天平仓避开了这个"死亡区间"，使S4从年化-2.3%变成+4.1%。
> 回测验证：该规则使组合夏普从1.47提升到1.67（+14%），卡玛从3.19提升到3.97（+24%）。

## 六、保证金约束体系

### 6.1 三层保证金上限

| 约束 | 上限 | 计算基础 |
|------|------|---------|
| 组合总保证金 | 50% NAV | 所有策略卖腿保证金之和 |
| S1保证金 | 25% NAV | S1卖腿保证金之和 |
| S3保证金 | 25% NAV | S3卖腿保证金之和 |

### 6.2 约束检查时机

1. **新开仓时**：先检查是否超限，超限则跳过
2. **止盈重开时**：重开前检查S策略独立上限和组合总上限
3. **开仓优先级**：S3优先于S1（同一品种同一天）

### 6.3 保证金计算规则

- **只计卖腿**（sell role）的保证金
- 买腿（buy/protect role）不计入保证金占用
- S4为纯买入策略，不占用保证金

## 七、执行价格模型

回测使用T+1日VWAP作为执行价格，更接近真实交易情况：

```
执行价格 = T+1日 (High + Low + Close) / 3
如果T+1日该合约无成交数据，退化为T日收盘价
```

| 项目 | 说明 |
|------|------|
| 信号生成 | T日收盘后，使用T日数据 |
| 执行价格 | T+1日VWAP = (H+L+C)/3 |
| T+1价格可用率 | 约70% |
| 退化方案 | T+1无数据时用T日收盘价 |

## 八、盈亏计算

### 8.1 每日盯市（MTM）

```
卖腿 daily_pnl = (prev_price - cur_price) × multiplier × quantity
买腿 daily_pnl = (cur_price - prev_price) × multiplier × quantity
```

### 8.2 NAV计算

```
NAV = CAPITAL + 累积daily_pnl
```

不扣除手续费（手续费仅记录在trades中，不影响NAV曲线）。

### 8.3 总盈亏

```
卖腿 total_pnl = (open_price - cur_price) × multiplier × quantity
买腿 total_pnl = (cur_price - open_price) × multiplier × quantity
```

### 8.4 止盈百分比

```
profit_pct = (open_price - cur_price) / open_price  （仅对卖腿有意义）
```

## 十、每日运营流程

### 10.1 收盘后流程（T日15:15后执行）

```
1. 获取T日所有20品种的期权收盘数据
2. 更新所有持仓的当前价格（MTM）
3. 计算当日NAV
4. 检查每个卖腿持仓：
   a. 若 profit_pct >= TP 且 DTE > 5 → 生成平仓订单
   b. 若 DTE <= 1 → 生成到期平仓订单
5. 对止盈平仓且 DTE > 10 的品种，尝试重开：
   a. 选择新合约
   b. 检查保证金约束
   c. 生成开仓订单
6. 检查所有品种是否有新的开仓信号（DTE≈35）：
   a. S3优先开仓
   b. S1次优先
   c. S4最后（仅限S4品种）
7. 输出订单CSV文件
8. 更新持仓状态文件
```

### 10.2 次日执行（T+1日开盘后）

按订单CSV中的指令，以参考价附近挂单执行。

## 十三、已知局限

1. **前视偏差风险**：批量回测中 `open_on` 字典预计算了所有到期日的开仓日（使用了未来数据），逐日回测系统通过实时检测消除此偏差
2. **单行权价假设**：回测框架只在每个品种的单一行权价上建仓，实盘可分散到多个行权价
3. **保证金公式简化**：使用CFFEX公式统一计算所有品种，与各交易所实际公式可能有差异
4. **手续费不影响NAV**：手续费仅记录不扣减，实际影响较小（约占收益<1%）
5. **数据覆盖**：缺少广期所碳酸锂/多晶硅/工业硅等品种
