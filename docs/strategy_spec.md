## 一、策略总览

本策略是以卖出深虚值期权收取权利金为核心的低频组合策略，包含三个子策略：

| 子策略 | 名称 | 角色 | 贡献 |
|--------|------|------|------|
| **S1** | 垂直价差保护卖权 | 核心收益层 | Theta衰减收入，垂直价差限制尾部亏损 |
| **S3** | 裸比例价差 + 应急蝶式保护 | 收益增强层 | 裸比例价差增厚收益，应急时加保护腿限制尾部 |
| **S4** | 尾部对冲 | 保险层 | 买入深虚值期权，正Gamma对冲极端风险 |

**核心逻辑**: 在动态品种池（按持仓量排序Top20）上，持续卖出深虚值期权（|delta|<0.15），利用时间衰减（Theta）获利，同时通过保护腿结构和尾部对冲控制风险。

## 三、S1 垂直价差保护卖权

### 3.1 策略结构

双卖（Put+Call），每方向独立执行：
- **卖腿**（sell）：卖出深虚值期权，收取权利金
- **保护腿**（buy）：买入更接近平值的同类期权，限制最大亏损
- 比例：卖腿N手 : 保护腿N/2手（2:1）

### 3.2 开仓条件

**触发时机**：每日实时检查，当某品种存在DTE最接近35天的到期月，且今天比明天更接近target时触发。无前视偏差。

**前置检查**：
1. 品种IV分位 ≤ 80%（否则暂停）
2. 组合Cash Vega ≤ 1.5%（否则暂停，v2.1新增）
3. 组合总保证金 < 50% NAV
4. S1保证金 < 25% NAV

### 3.3 合约选择

**卖腿**：|delta| < 0.15，权利金 ≥ 0.5，选资金效率最高（权利金×乘数/保证金）
**保护腿**：|delta| < 0.25，比卖腿更接近平值，选|delta|最大

### 3.4 手数计算

```
卖腿手数 = max(1, floor(NAV × 0.02 / 2 × iv_scale / 单手保证金))
保护腿手数 = max(1, floor(卖腿手数 / 2))
iv_scale = 0.5 if IV分位 > 75% else 1.0
```

### 3.5 止盈

- 卖腿浮盈 ≥ 40% 且 DTE > 5 → 平仓整组
- 若 DTE > 10 且 IV分位 ≤ 80% → 立即重开

### 3.6 到期平仓

DTE ≤ 1 → 平仓整组

## 四、S3 裸比例价差 + 应急蝶式保护

### 4.1 策略结构

双卖（Put+Call），每方向独立执行：
- **买腿**（buy）：1手，OTM 5%（浅虚值，正Gamma主腿）
- **卖腿**（sell）：2-3手，OTM 10%（深虚值，覆盖买腿成本）
- **保护腿**（protect）：仅在极端行情时临时加入，OTM 15%，手数=卖腿手数

常态结构为裸比例价差（买+卖，不带保护腿），零成本进场。
仅当卖腿OTM%降至5%以下（接近平值）时，临时买入保护腿转为蝶式。

### 4.2 开仓条件

与S1相同。**优先级高于S1**：同一品种同一天，先开S3再开S1。
同品种同方向同一时间只允许一组S3卖腿持仓。

### 4.3 合约选择（按OTM%筛选）

**买腿**：OTM 3%-7%，选最接近5%的合约，权利金≥0.5
**卖腿**：OTM 7%-13%，比买腿更虚值，选权利金最高，权利金≥0.5
**保护腿**（应急时）：OTM 12%-20%，比卖腿更虚值，选最接近15%，权利金≥0.1

OTM% = |1 - strike / spot_close| × 100

### 4.4 手数计算（灵活比例 + 零成本进场）

```
先尝试1:2比例，若净权利金不够覆盖买腿成本再尝试1:3比例。
buy_qty = max(1, floor(NAV × margin_per / 2 × iv_scale / (sell_margin × ratio)))
sell_qty = buy_qty × ratio
零成本约束：sell收入 - buy成本 ≥ -buy成本 × 0.3（容忍度30%）
两种比例都不满足时放弃该方向开仓。
```

### 4.5 应急保护机制

每日检查所有S3卖腿持仓：
- 当卖腿OTM%降至5%以下（标的价格向卖腿方向大幅移动）→ 触发应急保护
- 买入保护腿（OTM 15%），手数=卖腿手数，将裸比例价差转为蝶式
- 无合适保护合约时平仓整组（风控兜底）

### 4.6 止盈

- 卖腿浮盈 ≥ 30% 且 DTE > 5 → 平仓整组（含保护腿如果有）
- 若 DTE > 10 且 IV分位 ≤ 80% → 重新开仓（仅裸比例价差，不带保护腿）

## 五、S4 尾部对冲

### 5.1 策略结构

纯买入策略：买入最深虚值Put + Call（宽跨式）

### 5.2 参与品种

跟随动态品种池（当月所有活跃品种）。预算按品种数动态分配。

### 5.3 合约选择

最深虚值（moneyness最小的Put / 最大的Call），权利金 ≥ 0.1

### 5.4 手数计算

```
每方向预算 = NAV × 0.005 / 品种数 / 2
手数 = min(max(1, floor(预算 / (权利金 × 乘数))), 5)
```

### 5.5 平仓条件（v2更新）

- **DTE < 10 时平仓**（v2改进，避免Theta加速衰减区间）
- DTE ≤ 1 到期平仓
- fallback：持仓超过30天（无DTE数据时的安全网）

> v1用"持仓15天"，v2改为"DTE<10"。开仓DTE=35，持有到DTE=10（约25天），
> 比v1多10天保护窗口，且精确避开了Theta加速衰减的最后10天。

## 七、风控体系（v2新增）

### 7.1 组合Greeks监控

| 指标 | 日常 | 预警 | 硬止损 |
|------|------|------|--------|
| Cash Delta | <10% NAV | >15% | >20% |
| Cash Gamma | <20% NAV | >30% | >40% |
| Cash Vega | <1% NAV | >1.5% | >2% |
| Cash Theta | 正值 | <0 | - |

**Cash Vega触发动作**：
- >1.5%：暂停所有新开仓
- >2%：按Vega贡献从大到小平仓至<1.5%
- >3%：全部平仓（极端防线）

### 7.2 IV环境过滤

- IV分位 > 75%：新开仓手数减半
- IV分位 > 80%：暂停该品种S1/S3新开仓和止盈重开（S4不受限）

### 7.3 保证金阶梯

| 阈值 | 动作 |
|------|------|
| >40% | 关注 |
| >45% | 预警，暂停新开仓 |
| >50% | 警告，评估减仓 |
| >55% | 强制减仓至45%以下 |

### 7.4 压力测试（每日输出）

每日计算完整场景损益矩阵（18个场景）：

| 标的变动 | IV+5vol | IV+10vol | IV+20vol |
|---------|---------|----------|----------|
| -5% | stress_s-5_iv5 | stress_s-5_iv10 | stress_s-5_iv20 |
| -3% | stress_s-3_iv5 | stress_s-3_iv10 | stress_s-3_iv20 |
| -1% | stress_s-1_iv5 | stress_s-1_iv10 | stress_s-1_iv20 |
| +1% | stress_s1_iv5 | stress_s1_iv10 | stress_s1_iv20 |
| +3% | stress_s3_iv5 | stress_s3_iv10 | stress_s3_iv20 |
| +5% | stress_s5_iv5 | stress_s5_iv10 | stress_s5_iv20 |

计算公式：
```
stress_pnl ≈ Cash_Delta × ΔS + Cash_Gamma × 0.5 × ΔS² + Cash_Vega × ΔIV
```

### 7.5 品种相关性监控（v2.1新增）

|------|------|------|
| 品种间相关性矩阵 | 每20天 | 任意对>0.7 |

## 八、执行模型（v2.1更新）

### 8.1 T+1分钟VWAP执行

```
T日15:15收盘后：
  1. MTM更新持仓 + 计算组合Greeks
  2. 检查止盈/到期 → 生成平仓订单（pending_closes）
  3. 检查新开仓信号 → 生成开仓订单（pending_opens）

T+1日开盘后：
  4. 执行平仓（用T+1日开盘后N分钟VWAP + 买卖价差）
  5. 执行开仓（用T+1日开盘后N分钟VWAP + 买卖价差）
  6. 手数受日均成交量10%约束
```

### 8.2 执行价格优先级

1. 分钟数据窗口VWAP（开盘后10分钟，精确窗口）
2. 分钟数据全天VWAP（窗口VWAP不可用时）
3. 日线 (H+L+C)/3（无分钟数据时）
4. T日收盘价（无T+1日数据时）

### 8.3 买卖价差模拟（v2.1新增）

执行价格在VWAP基础上加减半个买卖价差（spread_proxy）：
- 卖出开仓：执行价 = VWAP - spread_proxy / 2（卖方吃价差）
- 买入开仓：执行价 = VWAP + spread_proxy / 2（买方吃价差）
- 平仓方向相反

spread_proxy = 分钟级 (high - low) 的日均值，作为买卖价差的代理。

### 8.4 成交量约束（v2.1新增）

每笔开仓手数不超过该合约当日成交量的10%：
```
实际手数 = min(计算手数, 日均成交量 × 10%)
```

### 8.5 手续费

每手开+平 14元，计入NAV。

> 该手续费为偏保守假设。实际各交易所手续费：中金所约15元/手，大商所1-2元/手，
> 上期所2-5元/手，郑商所1-3元/手。统一按14元/手计算高于大部分商品期权的实际费率，
> 使回测结果在手续费维度上偏保守。

## 十、已知局限与改进方向

1. **保护腿比例2:1**：S1有一半裸卖敞口，极端行情下需监控裸卖部分的组合Delta
2. **S1/S3同品种叠加**：同品种可能同时有S1和S3卖腿，风险叠加
3. **品种集中度**：农产品占比可能过高，需监控板块集中度
4. **止盈只看卖腿**：未考虑保护腿亏损，整组净盈亏可能低于止盈阈值
5. **美式期权提前行权**：深虚值概率极低但非零，需监控moneyness穿越
6. **节假日减仓**：长假前应减仓，当前未在引擎中实现
7. **涨跌停处理**：未模拟涨跌停无法平仓的情况
