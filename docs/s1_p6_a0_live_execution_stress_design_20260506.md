# S1 P6 A0 实盘口径压力回测设计

## 1. 实验目的

P5 当前阶段显示，A0 即 `1.5x 权利金止损 + group close` 可能是最稳的止损主线。但 A0 是否可作为实盘候选，不能只看回测 NAV；必须单独验证止损成交口径。

本实验不改变 A0 的交易逻辑，只压力化止损执行：

- 不改变品种池。
- 不改变 P/C 和合约筛选。
- 不改变保证金预算。
- 不改变 `premium_stop_multiple = 1.5`。
- 不改变 `s1_stop_close_scope = group`。
- 只改变止损成交价、确认强度和低流动性成交折价。

## 2. 为什么要做执行压力

卖权策略的净收益可拆成：

```text
S1 net return
= Premium Pool
× Deployment Ratio
× Retention Rate
- Tail / Stop Loss
- Cost / Slippage
```

A0 主要改善的是 `Tail / Stop Loss`：价格一旦恶化就整组退出。但实盘中止损退出会同时影响：

- `Retention Rate`：假止损会降低权利金留存。
- `Tail / Stop Loss`：止损慢一两分钟，亏损可能扩大。
- `Cost / Slippage`：买平卖方期权时需要付 ask、冲击成本和追价。

因此 A0 的正式评价必须加入成交压力。

## 3. 当前 A0 交易规则

A0 配置：

```text
premium_stop_multiple = 1.5
s1_stop_close_scope = group
s1_layered_stop_enabled = false
```

`group` 定义为：

```text
S1_<product>_<option_type>_<expiry>_<open_date>
```

即同一品种、同一 P/C 方向、同一到期日、同一开仓日的一组卖腿一起平仓。

## 4. 代码口径修正

启动 P6 前修正一个实现口径：

原先 `_process_intraday_exits()` 只要完成盘中扫描就可能返回 True，从而让主循环跳过日频止损兜底。P6 中改为只有当持仓数量、手数、已实现 PnL 或手续费实际变化时才返回 True。

这个修正是 P6 控制组必须重跑的原因。

## 5. 实验分组

### P6-A0-codefix

用途：A0 原规则，在修复盘中退出返回值后重跑，作为 P6 控制组。

### P6-mild

轻度实盘压力：

- 止损买平额外滑点：`1.0%`
- 普通开平仓滑点：沿用基础配置
- 最小绝对滑点：`0.10`
- 止损确认：2 次观测
- 低流动性过滤：沿用基础配置

解释：模拟正常盘口中买平需要付更差价，但不是极端追价。

### P6-conservative

中度实盘压力：

- 止损买平额外滑点：`2.0%`
- 普通平仓滑点：`0.30%`
- 最小绝对滑点：`0.20`
- 止损确认：3 次观测
- 最小成交量：5 手
- group 成交量比例：20%

解释：模拟触发后需要等更多成交确认，且对低价期权加入更高 tick 成本。

### P6-severe

重度实盘压力：

- 止损买平额外滑点：`5.0%`
- 普通平仓滑点：`0.50%`
- 最小绝对滑点：`0.50`
- 止损确认：3 次观测
- 最小成交量：10 手
- group 成交量比例：50%

解释：模拟极端行情中卖方买平追价、流动性退化和低价合约 tick 放大的最坏可交易场景。

### P6-next-minute-high

下一分钟最高价成交压力：

- 触发：仍按当前分钟止损确认规则。
- 成交：不使用触发分钟 close，而使用下一分钟 high 作为卖方买平参考价。
- 额外止损滑点：`0%`，用于单独隔离“下一分钟最不利成交”的影响。
- 最小绝对滑点：`0`。

解释：模拟实盘发现止损、下单、排队和追价的延迟风险。该口径不应用作预测规则，因为下一分钟 high 是事后分钟 K 线信息；但它适合作为执行压力测试，用于衡量 A0 是否依赖过于理想的止损成交。

## 6. 主要观察指标

必须与 P6-A0-codefix 对齐共同截止日比较：

- NAV / 累计收益 / 年化收益。
- 最大回撤。
- Sharpe / Calmar。
- Vega PnL / Theta PnL。
- 止损次数和止损 PnL。
- `execution_slippage_cash`。
- 权利金留存率。
- 最差单日和止损簇集日期。
- P/C 和品种集中度是否因止损成交压力变化。

## 7. 判断标准

如果 P6-mild 和 P6-conservative 下 A0 仍显著优于其他止损结构，则 A0 可以升级为实盘模拟主线。

如果 mild 即显著劣化，则说明 A0 的回测优势主要来自过于乐观的止损成交，下一步应回到：

- 放宽止损倍数；
- 增加止损确认；
- 降低低流动性合约权重；
- 减少 group close 的误杀；
- 或用更保守的合约层选择降低触发率。

如果 severe 才明显劣化，这是可接受的，因为 severe 是压力场景，不是基准成交假设。
