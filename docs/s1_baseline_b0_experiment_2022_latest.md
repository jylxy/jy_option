# S1 B0 基准实验记录：2022 至最后可用数据日

## 1. 实验目的

本实验用于给 S1 卖权研究建立最朴素的长期基准。它不追求最优参数，而是回答：

```text
如果从 2022 年开始，只用全品种、次月、低 Delta、等权、持有到期的卖权规则，
在真实费用和真实保证金率下，原始卖方权利金到底能不能覆盖尾部风险？
```

后续所有 S1 改进版本都应与这个基准比较，而不是只和上一个复杂版本比较。

## 2. 配置文件

```text
config_s1_baseline_b0_all_products_stop25.json
```

核心规则：

- 只启用 S1，不启用 S3/S4/保护腿。
- 默认全品种池，品种上市且有数据后即可进入扫描。
- 只选择次月到期合约，即当日可交易到期序列中的第 2 个到期日。
- 只卖出 OTM 且 `abs(delta) <= 0.10` 的 Put/Call。
- 同一品种、同一方向、同一次月下所有符合 Delta 条件的合约都作为候选。
- 资金按当日可交易品种等权，再在品种内部按 P/C 和合约等权铺开。
- 组合目标保证金上限 50%，S1 自身保证金上限 50%。
- 不使用 IV/RV、降波、趋势、动量、forward vega、stress score 或组合集中度过滤。
- 不止盈、不主动滚仓、不做临近到期提前平仓。
- 到期日按内在价值结算，虚值归零。
- 唯一提前退出是权利金涨到开仓权利金 2.5x，且沿用当前盘中止损防跳变确认逻辑。
- 手续费和保证金率使用当前接入的期货公司/交易所表，滑点沿用成本版假设。

## 3. 回测区间

```text
start_date = 2022-01-01
end_date = 服务器交易日缓存中的最后可用交易日
```

正式运行前先用短区间校验订单和持仓是否符合 B0 定义。

## 4. 输出标签

```text
s1_baseline_b0_allprod_2022_latest
```

预期落盘文件：

- `output/nav_s1_baseline_b0_allprod_2022_latest.csv`
- `output/orders_s1_baseline_b0_allprod_2022_latest.csv`
- `output/diagnostics_s1_baseline_b0_allprod_2022_latest.csv`
- `output/report_s1_baseline_b0_allprod_2022_latest.md`

## 5. 必看诊断

跑完后必须检查：

- 实际回测截止日。
- 平均和最高保证金使用率，确认是否接近 50%。
- 开仓品种数、活跃品种数、活跃合约数。
- P/C 结构和单品种最大暴露。
- 止损次数、止损品种、止损成交价相对 2.5x 阈值的偏离。
- 到期虚值归零次数、到期实值次数、到期实值亏损。
- 年度、季度、月份收益。
- 2025 年 4 月和 7 月等升波阶段表现。
- Greek 归因中 theta、vega、delta、gamma、residual 的累计贡献。
- ETF、股指、商品三类市场分别的贡献。
