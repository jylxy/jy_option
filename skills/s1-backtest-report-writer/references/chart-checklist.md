# S1 报告图表深度解读清单

报告插图不只是附录。每张图都需要回答一个研究问题，并把图上的现象翻译成期权卖方语言：收益来源、风险补偿、尾部暴露、仓位效率和下一步实验。

每张图下必须包含：

- 怎么看：图的读法和关键观察维度。
- 图上读数：具体日期、峰值、低点、占比、斜率、跳变、异常点。
- 期权专家判断：它说明策略在 theta、vega、gamma、delta、尾部、P/C、流动性或保证金上有什么问题。
- 风险或口径疑点：这张图不能解释什么，是否可能受数据、成交、Greek 口径或绘图口径影响。
- 下一步验证：对应一个可执行拆解或对照实验。

| 图表 | 研究问题 |
|---|---|
| `01_nav_drawdown.png` | 收益是否平滑，回撤是否集中，是否达到目标收益/回撤；必须指出最大回撤发生区间、修复速度、是否是一两个事件主导。 |
| `02_margin_positions.png` | 是否真的用了仓位，还是规则导致低暴露；必须结合保证金、持仓数、活跃品种/合约判断“仓位提升是否有效”。 |
| `03_greeks_timeseries.png` | cash Greeks 是否长期偏向单侧，尾部前是否已预警；必须说明 delta 是否方向化、gamma 是否在到期前堆积、vega 是否在升波前过高。 |
| `04_pnl_attribution.png` | 收益来自 theta/vega 还是 delta 侥幸；必须判断 theta 是否覆盖 gamma、vega 是否为正、residual 是否过大。 |
| `05_daily_pnl_tail.png` | 左尾是否吞掉多月收益；必须比较最大亏损日与平均 theta 收益、手续费、止损簇集和最差 5% 日期。 |
| `06_premium_pc_structure.png` | 双卖是否实际变成单边方向仓；必须拆 P/C 手数、权利金、保证金和 PnL，而不只看手数。 |
| `07_vol_regime_exposure.png` | falling/low/high/post-stop 状态的仓位是否合理；必须回答降波期是否给足预算，高波期是否还在加仓，低波/VCP 是否过度暴露。 |
| `08_calendar_returns.png` | 哪些月份决定全年收益和回撤；必须指出收益是否依赖少数月份，以及升波月份和降波月份分别发生了什么。 |
| `09_product_share_top10.png` | 全品种扫描是否真实分散；必须判断 top10 是否过度集中、是否商品/ETF/股指失衡、是否存在尾部共振隐患。 |
| `10_order_action_summary.png` | 持有到期、止损、期末持仓分别占比；必须评估收益是来自正常到期归零还是止损后重开、是否存在“赚小亏大”。 |
| `11_close_event_timeline.png` | 止损是否簇集，是否需要冷却和重开规则；必须识别同品种/同板块连续止损和跳变止损。 |

额外要求：

- 若报告比较两个版本，例如 B0 vs B1，图表解读必须写“变化来自哪里”，不能只说谁更好。
- 若净值更好但 vega/gamma 更差，必须提示“绩效改善不等于卖波质量改善”。
- 若图上出现异常跳变，必须检查是否可能来自异常报价、缺失成交、到期处理、乘数或保证金口径。
- 若图表缺失，报告必须说明缺失影响了哪个判断，并建议补图或补 CSV。
